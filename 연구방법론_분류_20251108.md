# MAICE 연구의 방법론적 분류
**작성일**: 2025-11-08

---

## 🎯 **MAICE 연구의 특징**

### **1. 연구자 역할**
- ✅ **교사-연구자** (Teacher-Researcher)
- ✅ 직접 시스템 설계·개발
- ✅ 자신의 교실에서 실험
- ✅ 실천과 연구의 통합

### **2. 연구 과정**
- ✅ 이론(Dewey, Bloom) → 설계 → 구현 → 실험 → 평가 → 개선
- ✅ 반복적·순환적 과정
- ✅ 맥락 중심 (수학적 귀납법)
- ✅ 실용적 산출물 (MAICE 시스템)

### **3. 연구 목적**
- ✅ 이론 검증 (Dewey의 명료화)
- ✅ 실천적 해결책 제시
- ✅ 설계 원리 도출
- ✅ 교육 현장 개선

---

## 📚 **주요 연구 방법론 분류**

### **1. Design-Based Research (DBR)** ⭐⭐⭐
**설계 기반 연구 / 디자인 연구**

#### **특징**
- 이론과 실천의 통합
- 교육 현장의 복잡한 문제 해결
- 반복적 설계-실험-개선 과정
- 실용적 산출물 + 이론적 원리 도출

#### **MAICE 연구와의 일치도**: 95%

| DBR 특징 | MAICE 연구 | 일치 |
|----------|-----------|------|
| **이론 기반 설계** | Dewey + Bloom 이론 기반 | ✅ |
| **반복적 개선** | 예비조사 → 시스템 개발 → 실험 | ✅ |
| **실제 맥락** | 실제 교실, 실제 학생 | ✅ |
| **실용적 산출물** | MAICE 시스템 | ✅ |
| **이론 기여** | 명료화의 효과 검증 | ✅ |
| **다학제적** | 교육학 + 컴퓨터과학 + AI | ✅ |

#### **DBR의 핵심 절차**
1. ✅ **문제 분석**: 학생 질문의 질적 문제 (예비조사)
2. ✅ **설계 원리**: Dewey 5단계 + Bloom K1-K4
3. ✅ **프로토타입**: MAICE 시스템 개발
4. ✅ **실험**: 58명 A/B 테스트
5. ✅ **평가**: AI + 교사 평가
6. 🔄 **개선**: (향후 반복 가능)

---

### **2. Action Research (실행 연구)** ⭐⭐

#### **특징**
- 실천가가 자신의 실천 개선
- "계획-실행-관찰-반성" 순환
- 현장 문제 해결 중심

#### **MAICE 연구와의 일치도**: 70%

| 측면 | 일치 여부 |
|------|----------|
| **교사가 연구자** | ✅ 완전 일치 |
| **현장 중심** | ✅ 자신의 교실 |
| **문제 해결** | ✅ 질문 품질 개선 |
| **순환적 과정** | ⚠️ 1회 실험 (반복 제한적) |
| **실천 개선** | ✅ 교수법 개선 |

**차이점**:
- 실행 연구는 보통 더 짧은 순환 반복
- MAICE는 시스템 개발이 포함되어 더 복잡

---

### **3. Developmental Research (개발 연구)** ⭐⭐

#### **특징**
- 교육 프로그램/도구 개발
- 체계적 개발 과정
- 효과성 검증

#### **MAICE 연구와의 일치도**: 80%

| 측면 | 일치 여부 |
|------|----------|
| **산출물 개발** | ✅ MAICE 시스템 |
| **체계적 설계** | ✅ 이론 기반 |
| **효과 검증** | ✅ A/B 테스트 |
| **현장 적용** | ✅ 실제 교실 |

---

### **4. Mixed Methods Research (혼합 연구)** ⭐

#### **특징**
- 양적 + 질적 연구 통합
- 다각도 분석

#### **MAICE 연구의 혼합**

| 방법 | 내용 |
|------|------|
| **양적** | A/B 테스트, 회귀 분석, 통계 검정 |
| **질적** | 예비조사 385건 질문 분석 |
| **통합** | 루브릭 개발 (질적 기준 → 양적 점수) |

## 🎯 **최종 분류: Design-Based Research (DBR)**

### **근거**

1. ✅ **이론-실천 통합**
   - Dewey/Bloom 이론 → MAICE 설계
   - 실험 결과 → 이론 검증

2. ✅ **반복적 과정**
   - 예비조사(2024.5) → 시스템 개발 → 본 실험(2024.10-11)

3. ✅ **맥락 중심**
   - 수학적 귀납법이라는 특정 단원
   - 고등학교 2학년이라는 특정 맥락

4. ✅ **이중 목표**
   - 실용적: MAICE 시스템 (현장 활용 가능)
   - 이론적: 명료화 효과 검증 (학술 기여)

5. ✅ **설계 원리 도출**
   - 질문 명료화 중심 설계
   - 멀티 에이전트 구조
   - 교사 커스터마이징 가능

## 📖 **주요 DBR 선행 연구**

### **대표 학자**
- **Ann Brown** & **Allan Collins** (1992): DBR 개념 정립
- **Jan van den Akker** (1999): 교육과정 개발에 DBR 적용
- **Feng Wang & Michael J. Hannafin** (2005): DBR 방법론 체계화

### **DBR의 특징 (Wang & Hannafin, 2005)**

| 특징 | MAICE 적용 |
|------|-----------|
| **Pragmatic** (실용적) | ✅ MAICE 시스템 개발 |
| **Grounded** (이론 기반) | ✅ Dewey + Bloom |
| **Interactive** (상호작용) | ✅ 연구자-학생-시스템 |
| **Iterative** (반복적) | ✅ 예비조사 → 본 실험 |
| **Flexible** (유연) | ✅ 맥락에 맞게 조정 |
| **Integrative** (통합적) | ✅ 양적+질적 방법 |
| **Contextual** (맥락적) | ✅ 수학적 귀납법 |

## 📝 **논문에서 명시할 표현**

### **연구 방법 섹션**

```markdown
**연구 방법론**

본 연구는 설계 기반 연구(Design-Based Research, DBR) 방법론을 
채택하였다. DBR은 교육 이론을 기반으로 실용적 산출물을 개발하고, 
실제 교육 맥락에서 반복적으로 실험·개선하며, 설계 원리와 이론적 
기여를 동시에 도출하는 연구 방법이다(Collins et al., 2004; 
Wang & Hannafin, 2005).

본 연구의 DBR 과정은 다음과 같다:

1. **문제 분석 및 탐색**: 예비조사(2024.5)를 통해 학생 질문의 
   질적 문제 확인 (385건 분석)

2. **이론 기반 설계**: Dewey의 반성적 사고 이론과 Bloom의 
   교육목표분류학에 기반하여 MAICE 시스템 설계

3. **개발 및 구현**: 5개 AI 에이전트로 구성된 멀티 에이전트 
   시스템 구현

4. **현장 실험**: 실제 교실 맥락에서 무작위 배정 A/B 테스트 
   수행 (58명, 2024.10.21~11.1)

5. **평가 및 분석**: AI 자동 채점 + 교사 평가로 효과성 검증

6. **설계 원리 도출**: 질문 명료화 중심 설계의 효과성과 
   학습자 수준별 차별적 효과 확인
```
