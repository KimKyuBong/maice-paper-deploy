## 📋 작업 개요

### 문제 인식
- 3장(시스템 설계)와 4장(시스템 구현)에서 3계층 아키텍처 설명이 중복됨
- 에이전트 역할이 두 챕터에서 반복 설명됨
- 독자가 "또 같은 얘기네?"라고 느낄 가능성

### 해결 목표
- **3장**: 교육적 설계 (WHY, WHAT) - 교육 이론 중심
- **4장**: 기술적 구현 (HOW) - 코드와 기술 스택 중심
- 명확한 역할 분담으로 논문 완성도 향상

## ✅ 완료된 수정 사항

### 1. 4장 서론 수정 ✅

**Before**:
```markdown
3장에서 설명한 MAICE 교육 시스템 아키텍처를 실제로 구현한 
기술 스택, 계층별 역할, 통신 메커니즘, 배포 전략을 다룬다.
```

**After**:
```markdown
본 장은 **3장의 교육적 설계를 실제 작동하는 시스템으로 구현한 
기술적 측면**을 다룬다. 교육적 근거와 설계 원리는 3장을 참조하고, 
여기서는 기술 스택, 코드 구조, 배포 전략에 집중한다.
```

**효과**: 4장의 목적을 명확히 하고, 3장 참조를 명시

#### 4.1.2 교육적 역할 설명 → 기술 다이어그램

**Before**:
- 계층 1: "학생이 수식을 쉽게 입력하고..." (교육적 설명)
- 계층 2: "프론트엔드와 에이전트를 연결..." (기능 설명)
- 계층 3: "질문 분류, 명료화..." (교육 프로세스)

**After**:
```markdown
### 4.1.2 기술 아키텍처 다이어그램

(간결한 Mermaid 다이어그램)

**통신 프로토콜**:
- 프론트 ↔ 백엔드: HTTP REST API, SSE
- 백엔드 ↔ 에이전트: Redis Streams
- 백엔드 ↔ DB: SQLAlchemy ORM

**에이전트 구성** (자세한 교육적 역할은 3장 3.3절 참조):
| 에이전트 | 기술적 역할 | 평균 응답 시간 |
```

**효과**:
- 교육적 설명 제거
- 기술 프로토콜에 집중
- "3장 3.3절 참조"로 중복 방지
