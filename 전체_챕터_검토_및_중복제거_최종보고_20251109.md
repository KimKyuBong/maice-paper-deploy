## 💡 표와 본문의 올바른 관계

### 설계 원칙

| 요소 | 역할 | 내용 | 예시 |
|------|------|------|------|
| **표** | 정량 데이터 요약 | 통계, 점수, 비율, 개수 | 72.3%, 평균 1.832점, 184건 |
| **본문** | 정성적 설명 | 사례, 해석, 함의, 맥락 | ID 294 "근데 어떻게 증명한거야?" |
| **참조** | 표-본문 연결 | "[표 X]에서 확인할 수 있듯이..." | 자연스러운 흐름 |

### 나쁜 예 (중복)

```markdown
[표 1-3] 맥락 없는 질문: 72.3%, 184건

#### 1) 맥락 없는 개념 설명 요청 (학습맥락 1점: 72.3%, 184건)
전체 질문 중 72.3%가 학습맥락 평가에서 최저점(1점)을 받았으며, 
교사 평가 기준으로 학습맥락이 모든 교사로부터 1점을 받은 질문이 184건 발견되었다.
```
→ 표의 모든 수치(72.3%, 184건, 1점)를 본문에서 그대로 반복!

### 좋은 예 (역할 분담)

```markdown
[표 1-3] 맥락 없는 질문: 72.3%, 184건

[표 1-3]에서 확인할 수 있듯이, 학습 맥락 부재가 가장 심각한 문제였다. 
대표적 사례는 다음과 같다:

**맥락 부재 사례 (ID 294):** "근데 어떻게 증명한거야?"
- 어떤 증명인지, 어느 부분이 어려운지, 학습자 수준이 무엇인지 전혀 파악 불가
- AI도 맥락을 알 수 없어 추가 정보 요청만 가능
```
→ 표에 통계, 본문에 정성적 사례! 중복 없음!
