### ✅ 적절한 설명 (유지 권장)

대부분의 다이어그램 주변 설명은 **다이어그램을 보완하는 필수 정보**입니다:

1. **상세 설명** (다이어그램에 없는 정보)
2. **맥락 제공** (왜 이 구조인지)
3. **해석** (다이어그램의 의미)

## ⚠️ 중복 가능성 있는 부분 (3곳)

### 1. IV장 - 명료화 평가 로직 (경미한 중복) ⚠️

**위치**: `04-system-implementation.md` 라인 89-92

**다이어그램 내용**:
```
PASS → 최종 질문 생성
NEED_MORE → 횟수 확인
< 3회 → 추가 명료화
≥ 3회 → 현재 정보로 진행
```

**다이어그램 뒤 텍스트**:
```markdown
**평가 기준**:
- **PASS**: 원본 질문의 의도가 명확해짐
- **NEED_MORE**: 여전히 모호하거나 추가 정보 필요
- **최대 3회 제한**: 무한 명료화 방지
```

**판단**: 
- `PASS`/`NEED_MORE` 의미는 다이어그램에 표시됨
- 하지만 **구체적 기준** ("의도가 명확해짐", "여전히 모호")은 추가 정보
- **유지 권장** (중복이지만 교육적 목적상 명시 필요)

---

### 2. III장 - QC 분류 (경미한 중복) ⚠️

**위치**: `03-system-design.md` 라인 293-303

**다이어그램 내용**:
```
학생 질문 → QC 분류
  ├─ answerable → AG 답변생성
  ├─ needs_clarify → QI 명료화
  └─ unanswerable → 안내 메시지
```

**다이어그램 뒤 텍스트**:
```markdown
> **구현 상세**: 프롬프트 설계 및 코드는 4장 참조
```

**판단**:
- 다이어그램만으로 충분히 이해 가능
- 하지만 **참조 정보**는 유용함
- **유지 권장**

---

### 3. III장 - 명료화 프로세스 흐름 (경미한 중복) ⚠️

**위치**: `03-system-design.md` 라인 357-360

**다이어그램 내용**:
```
명료화 반복 (최대 3회)
→ 정보 충분? 
  ├─ YES → 최종 질문 생성
  ├─ NO (3회 미만) → 반복
  └─ NO (3회 도달) → 최종 질문 생성
```

**다이어그램 뒤 텍스트**:
```markdown
**판단 기준**:
- **PASS**: 원본 질문의 의도가 명확해지고, 답변 생성에 필요한 정보 확보
- **NEED_MORE**: 원본 질문의 의도가 여전히 불분명하거나, 추가 정보 필요
- **최대 3회 제한**: 3회 명료화 후에도 불충분하면 현재 정보로 답변 생성
```

**판단**:
- IV장과 **동일한 내용 중복**
- 하지만 III장(설계)과 IV장(구현) 각각에서 설명 필요
- **유지 권장**

## 🎯 최종 권장사항

### 삭제 권장: **없음** ❌

모든 텍스트가 다음 중 하나에 해당:
1. **보충 정보** (다이어그램에 없는 상세)
2. **맥락 제공** (왜, 무엇을 위해)
3. **해석** (의미와 시사점)

### 유지 권장: **전부** ✅

경미한 중복(3곳)도 **교육적 목적**상 필요:
- 독자가 다이어그램 없이 텍스트만 읽어도 이해 가능
- 석사 논문은 "명확성 > 간결성"
