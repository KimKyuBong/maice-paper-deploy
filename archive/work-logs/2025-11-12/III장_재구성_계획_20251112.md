# III장 "설계-구현" 패턴 재구성 계획

**목표**: 각 컴포넌트별로 설계→구현 흐름으로 재구성

---

## 📋 현재 구조

```
III. MAICE 시스템 설계 및 구현 (718줄, 35.9p)

1. 예비 연구 (설계 근거)
2. 설계 철학 (교육 이론)
3. 시스템 아키텍처 (3계층)
4. 에이전트 설계 (QC, QI, AG, LO, FT)
5. 시스템 구현 (프롬프트)
```

**문제**: 설계 → 구현이 분리되어 흐름 단절

---

## 🎯 재구성 후 구조

```
III. MAICE 시스템 설계 및 구현 (718줄, 35.9p)

1. 예비 연구: 문제 분석 및 설계 근거
   - 예비조사 385건
   - 핵심 발견 4가지
   - MAICE 설계 반영 (유지)

2. 시스템 설계 개요
   - 가. 설계 철학: 명료화 중심 학습
   - 나. 시스템 아키텍처: 3계층 구조
   - 다. 질문 처리 파이프라인

3. 에이전트 설계 및 구현
   - 가. Question Classifier (QC)
     (1) 설계: K1-K4 분류 로직
     (2) 구현: 프롬프트 전문
   
   - 나. Question Improver (QI)
     (1) 설계: Dewey 5단계 기반 명료화
     (2) 구현: 평가 로직 + 프롬프트
   
   - 다. Answer Generator (AG)
     (1) 설계: K1-K4별 답변 차별화
     (2) 구현: K1-K4 템플릿 + 프롬프트
   
   - 라. Learning Observer (LO)
     (1) 설계: 학습 관찰 및 요약
     (2) 구현: 프롬프트
   
   - 마. Free Talker (FT)
     (1) 설계: 대조군 (즉시 답변)
     (2) 구현: 프롬프트

4. 통합 시스템 동작
   - A/B 테스트 배정
   - 데이터 수집 구조
```

---

## 🎁 효과

1. **논리적 흐름**
   - 각 에이전트: 설계→구현 연속
   - 독자 이해도 향상

2. **중복 제거**
   - 설계/구현 분리 불필요
   - 더 간결

3. **일관성**
   - 모든 에이전트 동일 패턴
   - 체계적
