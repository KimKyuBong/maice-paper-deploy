# MAICE 논문 설계도

## 논문 개요
**제목**: LLM 프리패스 방식 대비 우수한 학습 효과를 제공하는 AI Agent 시스템 설계 및 개발: 고등학교 2학년 수학적 귀납법 단원 중심으로

**핵심 주제**: MAICE agent 시스템이 일반적인 LLM 프리패스보다 학습 효과 및 경험이 우수함을 입증

---

## 논문 구조 및 흐름

```

├── 프리패스 방식의 한계
│   ├── 즉시 답변의 학습 효과 제한
│   ├── 질문 품질 낮음 (평균 2.28점/5점)
│   └── 학습 과정 깊이 부족
└── 해결 방안 제시
    ├── 명료화 프로세스 기반 AI agent 필요
    └── 수학적 귀납법 단원 특성 반영

연구 목적
├── 프리패스 대비 학습 효과성 검증
├── 명료화 기반 멀티 에이전트 시스템 개발
├── 학습 경험 개선 증명
└── 단원별 맞춤 설계

핵심 연구 문제 (2개)
├── RQ1: 학습 효과 우수성 (이해도, 문제해결, 질문 품질)
└── RQ2: 학습 경험 우수성 (만족도, 메타인지)
```

### 3장: MAICE 교육 시스템 아키텍처 (Educational System Architecture)
```
설계 철학: 명료화 중심 학습
├── Freepass 방식의 한계 분석
├── Bloom/Dewey 이론적 기반
├── 질문-분류-명료화-답변 파이프라인
└── 교육 목표 우선 설계

전체 아키텍처: 3계층 구조
├── 대화 인터페이스 (학생 경험 중심)
├── 관리 시스템 (세션, 인증, 데이터)
└── 에이전트 시스템 (교육 AI 로직)

5개 에이전트의 역할과 협업 ⭐
├── Classifier Agent
│   ├── K1-K4 질문 분류
│   ├── 3단계 게이팅 (응답가능/명료화필요/응답불가)
│   └── 명료화 질문 제안
├── Question Improvement Agent
│   ├── Dewey 5단계 반성적 사고 구현
│   ├── 3단계 차별화 명료화 전략
│   └── 대화형 질문 개선 프로세스
├── Answer Generator Agent
│   ├── K1-K4별 맞춤형 답변 구조
│   ├── 간결함/관계/절차/메타인지 훈련
│   └── 교육과정 표준 준수
├── Observer Agent
│   ├── 학습 진도 자동 추출
│   ├── 어려움 영역 감지
│   └── 교사 대시보드 제공
└── FreeTalker Agent (A/B 테스트 대조군)
    ├── 즉시 답변 방식
    └── 실험 통제 설계

질문 품질 평가 루브릭
├── K1-K4 지식 분류 기준
├── 12개 세부 평가 항목 (15점 만점)
└── 교육학적 근거 통합

명료화 프로세스 요약
├── 평균 명료화 횟수: 1.8회 (3회 제한)
├── 명료화 성공률: 83.3%
└── 질문 품질 향상: 평균 +46%

기술 구현 (4장에서 상세)
├── 프론트엔드: SvelteKit, TypeScript
├── 백엔드: FastAPI, PostgreSQL
├── 에이전트: Python, Redis Streams
└── 인프라: Docker, Nginx
```

### 4장: MAICE 시스템 구현 (System Implementation)
```
기술 스택 선정
├── 프론트엔드: SvelteKit 2.0 + TypeScript + MathLive
├── 백엔드: FastAPI + PostgreSQL + Redis
├── AI 에이전트: Python + Gemini 2.5 Flash
└── 인프라: Docker Compose + Nginx

프롬프트 엔지니어링
├── Classifier Agent (7,364자)
│   ├── K1-K4 분류 로직
│   ├── 3단계 게이팅 구현
│   └── Dewey 5단계 기반 명료화 질문 생성
├── Question Improvement Agent (6,471자)
│   ├── 명료화 완료 평가 (PASS/NEED_MORE)
│   ├── 질문 유형 재분류
│   └── 3단계 차별화 전략 (1회/2회/3회)
├── Answer Generator Agent (7,554자)
│   ├── K1-K4별 답변 템플릿
│   ├── LaTeX 수식 규칙
│   └── 교육과정 표준 용어
├── Observer Agent (7,522자)
│   ├── 세션 제목 생성 (15자 이내)
│   ├── 학습 요약 (200자 이내)
│   └── 교사 대시보드 정보
└── FreeTalker Agent (9,790자)
    └── 즉시 답변 (대조군)

통신 메커니즘
├── Redis Streams 기반 비동기 통신
│   ├── QUESTION_STREAM: 백엔드 → 에이전트
│   └── RESPONSE_STREAM: 에이전트 → 백엔드
├── Server-Sent Events (SSE)
│   └── 실시간 답변 스트리밍
└── 에이전트 간 협업 오케스트레이션

배포 및 인프라
├── Docker Compose 기반 배포
│   ├── PostgreSQL 15 (데이터베이스)
│   ├── Redis 7 (메시지 브로커)
│   ├── Backend (FastAPI)
│   ├── Frontend (SvelteKit)
│   ├── Agents (멀티프로세스)
│   └── Nginx (리버스 프록시)
├── 성능 최적화
│   ├── Connection Pooling
│   ├── Rate Limiting
│   └── 에이전트 자동 재시작
└── 모니터링 및 로깅
    ├── 구조화된 JSON 로깅
    ├── Prometheus 메트릭 수집
    └── 프롬프트 로깅 (재현 가능성)

구현 결과 요약
├── 평균 응답 시간: 2.3초
├── 동시 세션 처리: 최대 120개
├── 시스템 가용성: 99.2% (12일간)
└── 일일 LLM API 호출: 평균 387건
```

## 핵심 논리 흐름

```

    ↓

```

## 주요 차별점

1. **프리패스 대비 우수성**: 즉시 답변이 아닌 명료화 프로세스 통한 학습 효과 입증
2. **이론적 기반**: 듀이의 반성적 사고 이론을 체계적으로 적용
3. **멀티 에이전트 아키텍처**: 5개 독립 에이전트의 협업 구조
4. **단원별 맞춤 설계**: 수학적 귀납법 특성을 고려한 특화된 시스템
5. **실증적 검증**: 교사 평가와의 높은 상관관계 및 통계적 유의성 확인

---
