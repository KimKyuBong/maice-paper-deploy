### 2. 각 에이전트에 구현 섹션 추가

**QC (Question Classifier)**:
```
#### 구현: 프롬프트 전략
- System 메시지 핵심 (Bloom K1-K4, Dewey 5단계)
- 구현 특징 (Dewey 전략, K1-K4 분류, 학생 친화적 톤)
- > **전체 프롬프트**: 부록 A 참조
```

**QI (Question Improver)**:
```
#### 구현: 명료화 평가 로직
- 평가 로직 (PASS/NEED_MORE)
- 구현 특징 (이진 판단, 3회 제한, 유형별 차별화)
- > **전체 프롬프트**: 부록 A 참조
```

**AG (Answer Generator)**:
```
#### 구현: K1-K4 템플릿 선택 로직
- 템플릿 구조적 차이 (K1: 간결, K2: 연결, K3: 절차, K4: 메타인지)
- 코드 구현 (template_name = f"answer_{knowledge_code.lower()}")
- > **전체 템플릿**: 부록 A 참조
```

**LO (Learning Observer)**:
```
#### 구현: 학습 요약 전략
- 요약 전략 (패턴 감지, 상태 요약, 교사 보고)
- 구현 특징 (세션 컨텍스트, 패턴 감지, 익명화)
- > **전체 프롬프트**: 부록 A 참조
```

**FT (Free Talker)**:
```
#### 구현: A/B 배정 로직
- 무작위 배정 (hash(student_id) % 2)
- 구현 특징 (가입 시 배정, PostgreSQL 저장)
- A/B 구조 (Agent: 28명, Freepass: 30명)
- > **전체 프롬프트**: 부록 A 참조
```
