# III장 "설계→구현" 패턴 재구성

**목표**: 각 에이전트별로 설계 직후 구현 설명 ✅

---

## 📋 현재 문제

```
## 3. 5개 에이전트의 역할과 협업
   - QC 설계
   - QI 설계
   - AG 설계
   - LO 설계
   - FT 설계

...

## 5. 시스템 구현 (200줄 떨어진 곳)
   - 프롬프트 관리
   - K1-K4 템플릿
   - 명료화 로직
   - 프롬프트 전문
```

**문제**: 독자가 QC 설계 → QI 설계로 넘어가면 QC 구현을 놓침 ❌

---

## 🎯 재구성 후 구조

```
III. MAICE 시스템 설계 및 구현 (718줄)

1. 예비 연구 (유지)
2. 시스템 개요 (유지)

3. 에이전트별 설계 및 구현
   
   가. Question Classifier (QC)
      (1) 설계: K1-K4 분류 + 3단계 게이팅
      (2) 구현: 프롬프트 전략 (Dewey 5단계)
   
   나. Question Improver (QI)
      (1) 설계: Dewey 기반 명료화
      (2) 구현: PASS/NEED_MORE 평가 로직
   
   다. Answer Generator (AG)
      (1) 설계: K1-K4별 답변 차별화
      (2) 구현: K1-K4 템플릿 선택 로직
   
   라. Learning Observer (LO)
      (1) 설계: 학습 패턴 관찰
      (2) 구현: 요약 전략
   
   마. Free Talker (FT)
      (1) 설계: 대조군 (즉시 답변)
      (2) 구현: 단순 프롬프트

4. 통합 시스템 동작
   - A/B 배정
   - 데이터 수집
```

---

## 🔄 이동 계획

### Step 1: 기존 5절 해체
- 프롬프트 관리 → QC/QI/AG 구현으로 분산
- K1-K4 템플릿 → AG 구현
- 명료화 로직 → QI 구현
- 프롬프트 전문 → 각 에이전트 구현

### Step 2: 각 에이전트에 통합
```
### 가. QC
  #### (1) 설계
    - 기존 3절-가 내용
  #### (2) 구현
    - 프롬프트 전문 (기존 5절에서 이동)
    - 핵심 전략만 (상세는 부록)

### 나. QI
  #### (1) 설계
    - 기존 3절-나 내용
  #### (2) 구현
    - 명료화 평가 로직
    - 프롬프트 전략

### 다. AG
  #### (1) 설계
    - 기존 3절-다 내용
  #### (2) 구현
    - K1-K4 템플릿 선택
    - 각 템플릿 구조적 차이
```
