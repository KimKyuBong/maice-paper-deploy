    수학 학습 대화 질 평가를 위한 분석적 채점기준 (Analytic Rubric)¶

    📋 루브릭 사용 지침¶

    • 평가 대상: 학생-AI 에이전트 간 수학 학습 대화 전사본
    • 평가 단위: 하나의 질문-답변 쌍(turn)
    • 점수 범위: 각 영역 1~5점 (총 15점 만점)
    • 이론적 기반: 듀이의 반성적 사고, 메타인지 이론, 비계설정, 형성평가

    참고: 본 문서는 turn(질문-답변 쌍) 단위의 분석적 채점기준을 설명합니다. 실제 자동 채점 파이프라인에서 사용하는 세션 단위 루브릭(최종 질문 품질 중심 평가)과 JSON 스키마, 과정 지표는 docs/부록_A_루브릭_상세설명서.md를 참조하세요.

    1️⃣ 학생 질문의 질 (Student Question Quality) [5점 만점]¶

    평가 차원 - 문제 상황의 명료성 (듀이 1-2단계: 문제 인식 및 정의) - 메타인지적 인식 (Flavell: 자신의 인지 상태 인식) - 맥락 정보의 완결성 (비계설정: ZPD 진단 가능성)

    점수 수준 구체적 기술 예시
    5점 탁월 - 학습 목표와 현재 상태 명확 제시
    - 자신의 이해/오개념 메타인지 표현
    - 구체적 수학 맥락(문제, 단계, 시도, 제약조건) 포함
    - 원하는 답변 형태·깊이 명시    		 "귀납법 증명에서 n=k+1일 때 좌변을 전개하면 1+3+...+(2k-1)+(2k+1)이 되는데, 여기에 귀납가정 k²을 어떻게 사용하는지 모르겠어요. 식 변형 과정만 단계별로 보여주세요."
    4점 우수 - 학습 맥락 & 어려움 구체 제시
    - 시도한 방법/막힌 지점 언급
    - 부분적 메타인지 표현
    - 답변 형태 선호 암시    		 "귀납법 문제에서 귀납 가정을 언제 쓰는지 헷갈려요. 제가 풀던 문제는 1+3+5+...+(2n-1)=n² 증명인데 n=k+1 단계에서 막혔어요."
    3점 보통 - 일반적 주제/개념 수준 질문
    - 구체 맥락 부족하나 학습 단원 명시
    - 자신의 어려움 인식
    - 메타인지 표현 제한적    		 "수학적 귀납법에서 귀납 가정을 어떻게 사용하나요?"
    2점 미흡 - 모호하거나 광범위한 질문
    - 맥락 정보 거의 없음
    - 단순 정보 요청
    - 이해 상태 표현 없음    		 "귀납법이 뭐예요?" / "이 문제 풀이 알려주세요."
    1점 매우 미흡 - 질문 의도·목표 불명확
    - 학습 맥락 전혀 없음
    - 문장 불완전/이해불가
    - 수학학습 무관    		 "???" / "모르겠어요." / "숙제 대신 해주세요."

    채점 시 고려사항 - 가산 요소: 오개념 명시, 이전 학습 연결, 대안적 접근 시도 언급
    - 감점 요소: 즉답 요구, 과도한 정보 누락, 비학습적 의도

    2️⃣ AI 답변의 질 (AI Response Quality) [5점 만점]¶

    평가 차원 - 비계설정의 적절성 (ZPD 내 지원, 점진적 힌트) - 형성평가 원리 적용 (피드백의 3요소: Feed Up/Back/Forward) - 반성적 사고 촉진 (듀이 3-5단계: 가설-검증-결론)

    점수 수준 구체적 기술 예시 특징
    5점 탁월 - 학생 질문 재해석, 학습목표 명료화 (Feed Up)
    - 현재 이해 진단 + 구체 피드백 (Feed Back)
    - 단계적 힌트/소크라테스식 질문
    - 다음 단계·확장 방향 제시(Feed Forward)
    - 중간 검증 지점 명시    		 "귀납 가정 k²을 사용하려면 먼저 n=k+1의 좌변에서 k 관련 항을 분리해야 해요.

    1단계: (1+3+...+(2k-1))+(2k+1)로 묶어보세요. 첫 부분이 귀납가정과 같은가요?

    2단계: [학생 답변 대기]

    이렇게 하면 k²+(2k+1)=(k+1)²로 정리됩니다. 다음엔 부등식 귀납법에도 도전해볼까요?"
    4점 우수 - 질문 의도 파악, 학습목표 제시
    - 학생 수준의 설명, 단계별 안내 일부
    - 이해확인질문 1~2개, 다음 단계 암시    		 "귀납 가정은 n=k+1 증명에서 k 관련 부분을 대체하는 데 사용해요.

    예시 문제에서 1+3+...+(2k-1) 부분이 k²이라고 가정했으니, 여기에 대입하면 k²+(2k+1)이 되죠.

    여기까지 이해되나요? 그럼 이게 (k+1)²과 같은지 확인해볼게요..."
    3점 보통 - 직접적 답변
    - 설명은 정확, 단계 안내 부족
    - 학생 참여 유도/이해확인/다음단계 제시 미흡    		 "귀납 가정을 사용하는 방법은 다음과 같습니다.
    n=k일 때 성립한다고 가정하고, n=k+1일 때를 증명할 때 그 가정을 대입합니다.
    [완전한 풀이 제시]
    이해되셨나요?"
    2점 미흡 - 질문과 어긋난 답변·과도히 일반적
    - 즉답 제공, 사고 기회 박탈
    - 학생 수준 고려 없음
    - 검증/후속 안내 없음    		 "답은 이렇습니다. [전체 풀이] 다른 질문 있나요?" / "교과서 47페이지를 보세요."
    1점 매우 미흡 - 질문과 무관
    - 수학적 오류
    - 학습 방해(전체 직접답 제공 등)
    - 교육적 의도 없음    		 "잘 모르겠네요." / "이건 너무 어려운데요?" / [숙제 전체 답안 제공]

    채점 시 고려사항
    - 가산 요소: 시각적 표현, 오개념 진단, 학생 언어로 재표현
    - 감점 요소: 과도한 정보, 일방적 설명, 학생 수준 미고려

    3️⃣ 대화 맥락의 일관성 (Conversational Context Coherence) [5점 만점]¶

    평가 차원 - 대화 연속성 (이전 발화 참조 및 연결) - 적응적 조정 (학생 반응에 따른 전략 수정) - 학습 경로 관리 (목표 지향, 인지부하 조절)

    점수 수준 구체적 기술 평가 지표
    5점 탁월 - 이전 대화 명시적 참조
    - 이해 변화 추적·전략 적응
    - 목표 향한 일관 진행
    - 요약/전환 통한 인지부하 관리
    - 세션 구조화(시작-중간-종료)    		 - 3회 이상 이전 발화 참조
    - 학생 오류/오개념 후속확인
    - 목표 대비 진행 점검
    - 전략 변경 명시
    4점 우수 - 직전 발화 반영
    - 설명 수준 조정
    - 전체 흐름 유지
    - 중요 요약 겸 제공    		 - 2회 이상 이전 발화 참조
    - 부분적 전략 조정
    - 난이도 일관 유지
    3점 보통 - 현재 Q&A 집중
    - 이전 내용 참조 제한
    - 학생 반응 반영 미흡
    - 흐름 유지되나 전략적 연결 약함    		 - 1회 정도 이전 내용 언급
    - 독립적 turn 진행
    - 학습 궤적 불분명
    2점 미흡 - 대화 단절, 반복/모순 설명
    - 학생 상태 변화 무시
    - 산만, 방향성 없음    		 - 이전 발화 참조 없음
    - 동일 질문에 상이 답변
    - 목표 이탈
    1점 매우 미흡 - 맥락 완전 무시
    - 논리 연결 없음
    - 학생 혼란 가중
    - 구조 부재    		 - 맥락없는 독립 발화
    - 주제 무작위 전환
    - 진전 없음

    채점 시 고려사항
    - 가산 요소: 세션 시작/종료 요약, 학습자 상태 변화 민감성, 적절한 페이싱
    - 감점 요소: 순환 반복, 목표 망각, 과도한 주제 이탈

    📊 종합 평가 및 해석¶

    총점 분포¶

    총점 평가 해석
    13~15점 우수 교육적으로 효과적, 반성적 사고·메타인지 발달 촉진하는 상호작용
    10~12점 양호 기본적 학습 목표 달성, 일부 개선 필요
    7~9점 보통 정보 전달 가능, 교육적 깊이 부족, 명료화/비계설정 필요
    4~6점 미흡 학습 효과 제한, 질문-답변-맥락 모두 개선 필요
    1~3점 매우 미흡 교육적 가치 극히 낮음, 시스템 재설계/학습자 교육 필요

    영역별 해석 가이드¶

    • 질문 점수 낮음 → 학생에게 질문 명료화 전략 교육 필요 ("MAICE의 QuestionImprovementAgent" 역할 강화)
    • 답변 점수 낮음 → AI의 비계설정·형성평가 알고리즘 개선 필요
    • 맥락 점수 낮음 → 대화 이력 추적·적응 메커니즘 보강 필요 ("ObserverAgent" 역할 강화)
