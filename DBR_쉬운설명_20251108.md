# Design-Based Research (DBR) 쉬운 설명
**작성일**: 2025-11-08

---

## 🎯 **DBR이란?**

**설계 기반 연구 (Design-Based Research)**

교육 문제를 해결하기 위해 **이론을 바탕으로 실제 산출물(도구, 프로그램 등)을 만들고**, 
**현장에서 직접 실험하면서 개선**하고, 그 과정에서 **설계 원리와 이론적 통찰을 얻는** 연구 방법

---

## 🏗️ **비유로 이해하기**

### **전통적 연구 = 건축가**
```
1. 설계도만 그린다
2. 이론적으로 "이렇게 지으면 좋을 것이다" 제안
3. 실제로 짓지는 않음
```

### **DBR = 건축가 + 시공자 + 거주자**
```
1. 설계도를 그린다 (이론 기반)
2. 직접 집을 짓는다 (산출물 개발)
3. 사람들이 실제로 살게 한다 (현장 실험)
4. 문제점을 발견하고 고친다 (반복 개선)
5. "이렇게 지으면 좋다"는 원리 도출 (설계 원리)
```

---

## 📊 **전통적 연구 vs DBR**

| 구분 | 전통적 실험 연구 | DBR |
|------|-----------------|-----|
| **목적** | 가설 검증 | 문제 해결 + 이론 개발 |
| **산출물** | 논문 | **논문 + 실용적 도구** |
| **맥락** | 통제된 실험실 | **실제 현장** |
| **과정** | 1회 실험 | **반복적 개선** |
| **이론** | 기존 이론 검증 | **새로운 설계 원리 도출** |
| **연구자** | 관찰자 | **참여자 (교사-연구자)** |

## 🔄 **DBR의 핵심 과정**

### **1단계: 문제 발견** 🔍
```
현장에서 실제 문제 발견
예: "학생들 질문이 너무 불명확해서 AI가 제대로 답변 못함"
```

### **2단계: 이론 탐색** 📚
```
관련 이론 찾기
예: "Dewey의 반성적 사고 이론이 도움될 것 같아!"
```

### **3단계: 설계 및 개발** 🏗️
```
이론 기반으로 실제 도구 만들기
예: MAICE 시스템 개발 (5개 에이전트)
```

### **4단계: 현장 실험** 🧪
```
실제 교실에서 사용
예: 58명 학생 대상 A/B 테스트
```

### **5단계: 평가 및 분석** 📊
```
효과 측정
예: AI + 교사 평가로 학습 효과 확인
```

### **6단계: 개선 및 원리 도출** 🔄
```
문제점 개선 + 설계 원리 정리
예: "명료화 중심 설계가 하위권에 효과적"
```

---

## 🎓 **DBR의 핵심 특징 5가지**

### **1. 실용성 (Pragmatic)** 💡
- 이론만 제시하는 게 아니라 **실제로 쓸 수 있는 것**을 만든다
- MAICE: 실제 작동하는 AI 시스템

### **2. 이론 기반 (Grounded)** 📖
- 마음대로 만드는 게 아니라 **교육 이론에 근거**
- MAICE: Dewey + Bloom 이론 적용

### **3. 반복적 (Iterative)** 🔄
- 한 번에 끝이 아니라 **계속 개선**
- MAICE: 예비조사 → 개발 → 본 실험 → (향후 개선)

### **4. 맥락적 (Contextual)** 🏫
- 실험실이 아닌 **실제 현장**에서
- MAICE: 실제 고2 교실, 수학적 귀납법 단원

### **5. 통합적 (Integrative)** 🤝
- 양적 + 질적, 이론 + 실천 **모두 통합**
- MAICE: A/B 테스트 + 질문 분석, 이론 검증 + 시스템 개발

## 🌟 **MAICE 연구를 DBR로 보면**

### **문제 발견**
```
📝 예비조사 (2024년 5월)
"학생 질문의 72.3%가 맥락 정보 없음"
"45.8%가 구조 불명확"
→ 질문 품질이 문제다!
```

### **이론 탐색**
```
📚 Dewey (1910): 반성적 사고 5단계
   1단계: 문제 인식
   2단계: 문제 정의
   3단계: 가설 형성
   4단계: 추론
   5단계: 검증

📚 Bloom (2001): 지식 차원 분류
   K1: 사실적 지식
   K2: 개념적 지식
   K3: 절차적 지식
   K4: 메타인지적 지식
```

### **설계 및 개발**
```
🏗️ MAICE 시스템
├─ QuestionClassifier (질문 분류)
├─ QuestionImprover (질문 개선 유도)
├─ AnswerGenerator (답변 생성)
├─ LearningObserver (학습 관찰)
└─ FreeTalker (자유 대화)

Dewey 5단계 → AI 프롬프트로 구현
Bloom K1-K4 → 질문 분류 기준
```

### **현장 실험**
```
🧪 무작위 배정 A/B 테스트
- 대상: 고2 학생 58명
- 기간: 2024.10.21~11.1
- 비교: Agent (명료화) vs Freepass (즉시 답변)
- 데이터: 283개 세션
```

### **평가**
```
📊 다각도 평가
- AI 3개 모델 자동 채점
- 교사 4명 평가
- QAC 루브릭 (40점 만점)
```

### **원리 도출**
```
💡 설계 원리 발견
1. 질문 명료화가 학습 효과 향상
2. 하위권 학생에게 특히 효과적
3. 멀티 에이전트 구조의 장점
4. 교사 커스터마이징 가능성
```

## 🎯 **왜 DBR을 선택했나?**

### **다른 방법의 한계**

❌ **순수 이론 연구**
```
"명료화가 중요하다" → 그래서?
실제로 쓸 수 있는 게 없음
```

❌ **순수 개발 연구**
```
MAICE 만들기만 함 → 효과는?
이론적 근거 부족
```

❌ **전통적 실험**
```
실험실에서만 → 현장에서는?
일회성, 개선 없음
```

✅ **DBR**
```
이론 + 개발 + 실험 + 개선 + 원리
모두 통합! 🎉
```

## 📚 **DBR의 역사**

### **1992년**: 탄생
- **Ann Brown** & **Allan Collins**
- "교육 연구가 현장과 동떨어져 있다"
- → 이론과 실천을 통합하자!

### **2000년대**: 확산
- 교육공학 분야에서 활발
- 학습 환경 설계 연구
- 교수 도구 개발 연구

### **2010년대~**: 주류화
- 교육학 주요 방법론으로 인정
- AI 교육 도구 연구에 적합
- "Teacher as Designer-Researcher"

## 🌍 **DBR의 장단점**

### **장점** ✅

1. **실용성**
   - 논문만이 아니라 실제 쓸 수 있는 것 생산
   - MAICE: 다른 교사도 사용 가능

2. **현장 관련성**
   - 실험실이 아닌 실제 교실
   - 결과를 바로 적용 가능

3. **이론 발전**
   - 새로운 설계 원리 도출
   - "명료화 중심 설계"의 효과 검증

4. **교사 역량**
   - 소비자 → 생산자
   - 연구 능력 향상

### **단점** ⚠️

1. **시간 소요**
   - 설계 + 개발 + 실험 → 오래 걸림
   - MAICE: 6개월 이상

2. **복잡성**
   - 여러 단계 관리 필요
   - 기술 + 교육학 + 통계 모두 필요

3. **일반화 한계**
   - 특정 맥락(수학적 귀납법)
   - 다른 단원에도 효과? 추가 연구 필요

4. **객관성 우려**
   - 연구자 = 개발자 = 교사
   - 편향 가능성 (→ 다각도 평가로 보완)

## 💬 **쉬운 비유**

### **DBR = 레시피 개발자**

```
1. 문제: "맛있는 김치찌개 만들고 싶다"

2. 이론: 요리책 연구 (한식의 원리)

3. 개발: 실제로 만들어본다
   - 재료 선택
   - 순서 정리
   - 불 조절

4. 실험: 가족들에게 먹여본다
   - "너무 짜"
   - "고기가 부족해"

5. 개선: 레시피 수정
   - 소금 줄이기
   - 고기 추가

6. 원리: "맛있는 김치찌개 레시피" 완성
   + "돼지고기 김치찌개는 고춧가루를 나중에"
```

**결과물**:
- 📝 레시피 (이론/원리)
- 🍲 실제 김치찌개 (산출물)
