### 테마 1: 학습 효과 변화 - 답변 수동 수용 → 능동적 도출

#### 3.1.1 Agent 모드: 구조적 이해와 원리 파악

**[S04]** - 학습 방식의 근본적 전환:
> "MAICE를 쓰기전에는 단지 AI한테 물어보고 **답만 가져다가 적은 느낌**이라면 MAICE를 쓰고난 후에는 **논리적인 개념까지 알게되어서 답까지 내가 도출**하고 MAICE한테 나의 답이 맞는지 확인하는 식으로 사용한다"

**변화의 궤적**: 답변 복사 → 개념 이해 → 스스로 도출 → 검증

동일 학생의 추가 응답:
> "MAICE랑 수학적 귀납법에 대해서 이야기를 하다가 원래는 그냥 **공식처럼 외우고 있던거**를 이제 **그 원리를 알게**되어서 더 기억에 남고 쉽게 사용할수있다"

**학습 패러다임 변화**: 암기식 → 원리 기반 이해

---

**[S05]** - 사고의 확장:
> "**다른 귀납적 풀이**를 알려드릴까요? 라고 물어봤었는데, **다른 풀이가 있다고 생각을 안했었다**"

> "한 풀이가 아닌 **여러 풀이가 더 생각**해볼수 있었다"

**발견**: 단일 사고 → 다각적 접근  
**계기**: AI의 대안 제시 질문

---

**[S15]** - 질문의 정밀화:
> "질문하는방식이 **범위가크게질문**했었다면 지금은 **상세하게 필요부분만 질문**할수있음"

실제 경험:
> "오일러 항등식설명을 부탁하니 **답변이 두루뭉실**해서 별로였는데 **프롬프트를 상세하게 주니** 정확한 답변이 나와 이해하기 쉬웠음"

**학습 경로**: 실패 (모호한 질문 → 모호한 답변) → 개선 (구체적 질문 → 정확한 답변)

---

**[S03]** - 문제 해결 효율성 증가:

**과정**: 포기 직전 → 반복 질문 → 이해 성공 → 성취감

**패러다임 전환**: 일방향 Q&A → 쌍방향 대화

### 테마 3: 메타인지 발달 - 자기 이해와 점검

#### 3.3.1 Agent 모드: 자기 검증 시스템 구축

**[S04]** - 검증 도구로서의 AI:
> "논리적인 개념까지 알게되어서 **답까지 내가 도출**하고 **MAICE한테 나의 답이 맞는지 확인**하는 식으로 사용한다"

**사용 패턴 변화**: AI에게 답 요구 → 스스로 도출 → AI로 검증

---

**[S20]** - 검증 능력 획득:
> "maice를 쓸 때 과제를 제출하기전 **이 답이 맞는지 검증하지 못했**지만 이제는 **쉽게 검증할수있게**되었다"

**Before**: 검증 불가 → **After**: 검증 가능

---

**[S34]** - 전략적 활용:
> "귀납 가정을 풀 때 **다시 한 번 더 확인하는 용도**로 사용하였습니다"

> "귀납 가정 **탬플릿을 구상하는 데에** 많은 도움이 되었습니다"

**활용법**: ① 확인 도구 ② 템플릿 구축

---

**[S06]** - 자기 발견의 도구 ⭐⭐⭐:
> "내가 **안다고 생각하고 넘어간 문제들도**, maice를 통해 **검증하다보면 내가 모르는 부분을 발견**하게 된다. 그래서 **더욱 깊은 공부**를 할 수 있게 되었다"

**메타인지의 새로운 수준**:
- **잘못된 확신 인식**: "안다고 생각" → 실제로는 모름을 발견
- **자기 점검 능력**: 검증 과정에서 자신의 이해 한계 파악
- **깊이 있는 학습**: 피상적 이해 → 깊은 이해로 전환

**Agent 모드의 검증 도구화**:
```
[S04] "나의 답이 맞는지 확인" (검증 요청)
[S20] "쉽게 검증할수있게" (검증 능력)
[S06] "모르는 부분을 발견" (자기 발견) ← 가장 깊은 수준
```

### 테마 4: 깨달음의 순간 - 학습의 결정적 전환점

#### 3.4.1 Agent 모드: 원리 발견과 다각적 접근

**[S04]** - 암기에서 이해로:
> "원래는 그냥 **공식처럼 외우고 있던거**를 이제 **그 원리를 알게**되어서 더 기억에 남고 쉽게 사용할수있다"

**전환점**: 암기 → 원리 이해

---

**[S05]** - 다양성의 발견:
> "**다른 귀납적 풀이**를 알려드릴까요? 라고 물어봤었는데, **다른 풀이가 있다고 생각을 안했었다**"

> "MAICE가 **여러 방향을 제시**해 주었다. -> 이렇게 하는건 어떨까요?"

**깨달음**: 문제는 여러 방법으로 풀 수 있다

---

**[S20]** - 구조의 이해:
> "n^n > 2^n을 증명하는 문제가 있었는데 **처음에는 어떤식으로 해야하는지 감도 안 잡혔**는데 maice를 사용하니 **간단하게 귀납가정을 정의 할수있게** 되었다"

**변화**: 막막함 → 명확한 증명 구조 파악

---

**[S51]** - 맞춤형 학습:
> "그냥 **내가 원하는 풀이 방식으로 계속 알려주**니까 점점 어떻게 해야할지 이해가 됐다"

**효과**: 개인화된 설명 → 점진적 이해

---

**[S06]** - Rubber Duck 효과 발견 ⭐⭐⭐:
> "AI에 **프롬포트를 입력하는 도중에 갑자기 이해**가 되기 시작했다. **내가 AI에게 나의 상황을 설명하다 보니**, 자연스럽게 **해결 방안이 떠올랐다**"

**메커니즘**: 명료화 요구 → 자기 설명 → 이해 발생  
**교육심리학**: **Self-Explanation Effect** (Chi et al., 1994)

**Agent의 간접 효과**:
- AI가 직접 답을 주지 않아도 학습 효과 발생
- 학생이 자신의 상황을 설명하려 하면서 스스로 이해
- 프로그래밍의 "Rubber Duck Debugging"과 동일한 메커니즘

---

#### 3.4.2 Freepass 모드: 힌트 기반 즉각적 깨달음

**[S39]** - 가장 상세한 "Aha Moment" 서술:
> "MAICE와 대화하면서 가장 기억에 남는 순간을 꼽으라면, 단연 **귀납법 문제로 한참을 씨름하다가 AI의 질문 덕분에 번뜩 깨달음을 얻었던 그 순간**입니다. 마치 **꽉 막혀 있던 머릿속에 갑자기 빛이 들어온 듯한 짜릿함**이었죠"

**감정**: "짜릿함", "빛이 들어온 듯"  
**메커니즘**: 막힌 사고 → AI 질문 → 번뜩이는 통찰

동일 학생의 맥락:
> "복잡한 부등식 증명 문제에서는 더욱 그랬죠"

---

**[S53]** - 최소 힌트의 힘:
> "귀납법 문제를 풀다가 막혔을 때 MAICE가 **두 줄짜리 힌트**를 주었어요. 그 힌트 덕분에 **문제 접근 방법이 떠올라** 스스로 풀 수 있었습니다"

**효과**: 최소한의 힌트 → 자기주도적 문제 해결

### 테마 6: 답변 방식 선호도 (A vs B) - 블라인드 실험의 핵심 발견

#### 3.6.1 설문 설계

- **A 방식**: AI가 바로바로 답을 알려줌 (Freepass와 유사)

- **⚠️ 중요**: 학생들은 자신이 사용한 모드를 모르는 블라인드 상태

#### 3.6.5 ⭐ 블라인드 실험의 핵심 발견

- 설문 설명을 읽고 B의 교육적 가치 인정

**함의**:
1. **객관적 평가**: 경험과 무관하게 교육적 가치 판단 가능
2. **보편적 인정**: Agent 접근의 우수성을 사용하지 않은 그룹도 인정
3. **블라인드 설계의 타당성**: 편향 없는 응답 확인

**갈등**:
- **교육적 가치 인식** ≠ **실제 사용 의도**
- 일부 학생은 B가 좋다고 인정하면서도 A를 쓸 것 같다고 응답
- 이유: 편의성, 속도

## 🔍 5. 핵심 발견 (Key Findings)

### 5.1 주요 발견

**1) 학습 효과의 다른 경로**
- **Agent**: 원리 이해, 구조적 접근, 다양한 풀이
- **Freepass**: 힌트 활용, 효율적 문제 해결, 빠른 피드백
- → **다른 경로로 유사한 결과 도달**

**2) 질문 능력 향상 (공통)**
- 모든 학생이 "질문을 구체화하는 방법" 학습
- Agent: 이론적 질문으로 발달
- Freepass: 전략적 질문법 습득

**3) 명료화 프로세스의 양면성**
- **긍정**: 사고 확장, 깊은 이해
- **부정**: 순환 참조 (2명), 상세 요구 부담 (2명)
- → **설계 개선 필요**

**4) ⭐ 블라인드 실험의 객관성 입증**
- **Freepass 사용자 4명 (50.0%)이 B 방식 (질문 기반) 선호**
  - 자신이 경험하지 못한 방식을 선호
  - 설문 설명만 보고 교육적 가치 판단
- **Agent 사용자 7명 (77.8%)이 B 방식 선호**
  - 경험이 교육적 가치 인식을 강화
  - Agent 경험자의 B 선호율이 더 높음
- → 블라인드 실험 설계의 타당성 확인

**5) 교육적 가치 vs 사용 편의성 갈등**
- 일부 학생: "B가 교육적으로 좋지만 A를 쓸 것"
- 이유: 속도, 편의성
- → UX 개선 필요

### 5.2 학술적 함의

**1) 삼각검증 (Mixed Methods)**
- 양적: QAC 점수 (차이 없음)
- 질적: 학습 경험 (다름)
- → 통합 해석 필요

**2) Dewey (1910)의 이론 지지**
- "문제 명료화"가 학습에 기여
- 순환 참조 등 설계 한계로 효과 미발현

**3) 블라인드 실험의 중요성**
- 편향 없는 객관적 평가
- 교육적 가치의 보편적 인정

```markdown
1) **학습 효과 변화**: 답변 수동 수용 → 능동적 도출
   - Agent: "답만 가져다가 적은 느낌 → 논리적 개념까지 알게되어 답까지 내가 도출" (S04)
   - Freepass: "답과 풀이만 얻는식 → 힌트를 먼저 얻고 내가 스스로 풀어 본 뒤 검증" (S53)

2) **질문 명료화 학습**: 모호함 → 구체성
   - Agent: "이거 뭐야 이거 말해줘 → 이론적인 부분까지 질문" (S04)
   - Freepass: "대충 던져줬을 때 → 자세하게 역할을 부여하고" (S31)

3) **메타인지 발달**: 자기 이해와 점검
   - Agent: "나의 답이 맞는지 확인하는 식으로" (S04)
   - Freepass: "내가 뭘 아는지 뭘 모르는지, 즉 메타인지를 좀 더 잘" (S31)

4) **깨달음의 순간**: 학습의 결정적 전환점
   - Agent: "공식처럼 외우고 있던거 → 그 원리를 알게" (S04)
   - Freepass: "꽉 막혀 있던 머릿속에 갑자기 빛이 들어온 듯한 짜릿함" (S39)

5) **부정적 경험**: 시스템의 한계
   - Agent: "순환 참조" (2명), "상세 요구 부담" (2명)
   - Freepass: "답변 길고 이해 어려움" (2명)

6) **답변 방식 선호도**: 교육 효과 vs 편의성

**⭐ 블라인드 실험의 핵심 발견**:
설문에서 A 방식(즉시 답변)과 B 방식(질문 유도)을 설명한 후 선호도를 조사한
결과, Freepass 사용자 중 4명(57%)이 자신이 경험하지 못한 B 방식을 선호한다고
응답하였다. 학생들의 구체적 이유는 다음과 같다:

- "질문자가 생각해야지 수학 문제를 조금 더 이해를 잘할 수 있을 것" (S31)
- "AI가 질문을 던져 스스로 생각하게 한 뒤 답을 확인하면 이해가 깊어지고 문제 
   해결 능력이 향상" (S53)
- "AI와의 대화가 단순히 문제 해결을 넘어 학습과 성장의 과정이 되기를 바랍니다" 
  (S39)

이는 명료화 프로세스를 직접 경험하지 않은 학생들도 그 교육적 가치를 인정함을
보여주며, 블라인드 실험 설계의 타당성을 확인시켜준다.

반면 Agent 사용자 1명은 "어렵고 이해가 안가는걸 물어봐도 자꾸 질문을 줘서
불편한 경험" (S08)으로 A 방식을 선호하여, 현재 명료화 프로세스의 UX 개선
필요성을 보여준다.
```

### Discussion 섹션

**"8.X 양적-질적 통합 해석 및 블라인드 실험의 함의"**

```markdown
양적 분석(QAC 점수)과 질적 분석(서술형 응답)을 통합한 결과, 흥미로운 패턴이
발견되었다.

**양적 결과**: Agent vs Freepass 학습 효과 차이 없음 (p>.05)

**질적 결과**: 학습 '경험'은 다름
- Agent: "원리를 알게", "여러 풀이", "논리적 개념" → 구조적 이해
- Freepass: "힌트로 스스로 풀기", "빠른 피드백" → 효율적 문제 해결

이는 두 모드가 **다른 경로로 유사한 학습 결과**에 도달함을 시사한다.
Agent는 Dewey(1910)의 "문제 명료화" 과정을 통해, Freepass는 직접 지원을
통해 학습을 촉진한다.

**블라인드 실험의 중요한 발견**:
학생들은 자신이 어떤 모드를 사용했는지 모르는 상태에서 설문에 응답하였다.
이러한 블라인드 설계 하에서, Agent 경험자의 B 선호율(77.8%)이 Freepass(50.0%)보다 
높았다. 이는 **명료화 프로세스를 직접 경험함으로써 그 교육적 가치 인식이 강화됨**을 
보여준다.

동시에 Freepass 사용자의 절반(50.0%)이 자신이 경험하지 못한 B 방식을 선호한다는 
사실도 주목할 만하다. 이들은 실제로 즉시 답변 방식만 경험했음에도 불구하고, 
설문에서 제시된 두 방식의 설명을 읽고 질문 기반 접근의 교육적 가치를 인정한 것이다.

학생 24.017은 "AI와의 대화가 단순히 문제 해결을 넘어 학습과 성장의 과정이
되기를 바랍니다"라고 표현하며, 명료화 접근의 장기적 교육적 가치를 명시적으로
언급하였다.

이는 Agent 모드의 명료화 프로세스가 "경험하지 않아도 그 필요성이 인정되는"
교육적 타당성을 지님을 시사한다. 다만, Agent 사용자 1명이 "자꾸 질문을 줘서
불편" (S08)하다며 A 방식을 선호하고, 2명이 "순환 참조" 문제를 보고한 것은
현재 구현의 UX 문제로 인해 명료화 프로세스의 이점이 충분히 발현되지 못했을
가능성을 보여준다.

일부 학생들은 B 방식이 "발전에 더 도움이 될 것 같지만" 실제로는 "빠른 답변"을
위해 A를 쓸 것 같다고 응답하여 (S28, S09), 교육적 가치 인식과 실제 사용
의도 사이의 간극을 보여주었다. 이는 순환 참조 방지, 상세 요구 완화 등 UX
개선이 이루어진다면 Agent 모드의 잠재력이 더욱 발현될 수 있음을 시사한다.
```
