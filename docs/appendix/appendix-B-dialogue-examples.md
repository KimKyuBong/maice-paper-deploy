## 사례 1: 개념 구체화 요청형 (Choice-Based Clarification)

**세션 ID**: 124  
**명료화 유형**: 선택지 제공형  
**학생 수준**: 중간고사 객관식 47.2점 (중하위권)

### 초기 질문 (모호함)

```
[학생] "피보나치 수열을 수학적 귀납법으로 풀어저"
```

**문제점**: 
- 무엇을 "푸는" 것인지 불명확

```
         예를 들어:
         1) 일반항 공식
         2) 항들의 합 공식
         3) 항들 간의 관계
         
         이 중 어떤 부분이 궁금하신가요?"
```

**[학생 응답 1]**

```
[학생] "피보나치 수열을 수학적귀납법으로 풀어"

```

```
         - 'F(n) ≤ 2^n' (부등식 증명)
```

```
n=2일 때: F(2) = 1

[귀납 가정]
n=k, n=k+1일 때 성립한다고 가정

[귀납 단계]
```

## 사례 2: 구체적 어려움 파악형 (Problem-Focused Clarification)

**세션 ID**: 75  
**명료화 유형**: 어려움 지점 확인형  
**학생 수준**: 중간고사 객관식 52.1점 (중위권)

### 초기 질문

```
[학생] "수학적 귀납법으로 k, k+1일때 성립한걸 보인게 왜 증명이 되는거야?"
```

**좋은 점**: 
- 핵심 개념(귀납 단계)에 대한 의문 표현
- 이해하지 못한 부분 명시

**문제점**:
- "왜 증명이 되는가"의 의미가 다의적 (논리적 원리? 역사적 배경?)

---

### 명료화 질문

```
[MAICE] "좋은 질문입니다! '왜 증명이 되는가'에 대해 두 가지 관점으로 설명드릴 수 있습니다:

1) **논리적 원리**: n=k일 때 성립하고 n=k+1일 때도 성립하면, 
   도미노처럼 모든 자연수에 대해 성립하는 논리적 구조를 알고 싶으신가요?

2) **수학적 엄밀성**: 왜 이 방법이 수학적으로 타당한지 
   증명 체계를 이해하고 싶으신가요?

어떤 부분이 더 궁금하신가요?"
```

---

### 학생 후속 질문

```
[학생] "도미노 이론을 누가 만든거야?"
```

(명료화를 통해 학생의 진짜 궁금증이 드러남!)

---

### MAICE 답변

```
[MAICE] "'도미노 비유'는 수학적 귀납법을 이해하기 쉽게 설명하기 위한 
         교육적 비유입니다. 정확히 누가 처음 만들었는지는 명확하지 않지만...

[도미노 비유 설명]
- 첫 번째 도미노를 쓰러뜨림 = n=1일 때 성립
- k번째 도미노가 쓰러지면 k+1번째도 쓰러짐 = 귀납 단계
- 결과: 모든 도미노가 쓰러짐 = 모든 자연수에서 성립

이제 실제 증명 예시를 보여드릴까요?"
```

---

### 연속 학습 (심화)

```
[학생] "1+2+2^2+...+2^(n-1)=2^n-1 이걸 수학적 귀납법으로 증명해줘"
```

(명료화를 통해 원리를 이해한 후, 실제 증명 문제로 진입!)

- 개념 이해 → 원리 질문 → 실제 적용

## 사례 3: 증명 단계 구체화형 (Proof-Step Specification)

**세션 ID**: 139  
**명료화 유형**: 단계별 어려움 파악  
**학생 수준**: 중간고사 객관식 31.2점 (하위권)

### 초기 질문

```
[학생] "5이상의 모든 자연수 n에 대해, n^2 < 2^n 임을 
        수학적 귀납법을 이용하여 증명하시오"
```

(문제를 그대로 제시)

---

### MAICE 답변 (완전한 증명 제공)

```
[MAICE] 
[기저 단계]
n=5일 때: 5^2 = 25 < 2^5 = 32 ✓

[귀납 가정]
n=k일 때 k^2 < 2^k 성립한다고 가정

[귀납 단계]
n=k+1일 때도 성립함을 보이면...
(k+1)^2 < 2^(k+1)

[상세한 증명 과정]
```

---

### 학생의 어려움 표현

```
[학생] "$(k+1)^2 < 2^{k+1}$ 이려면 충분히 $2^k + 2k + 1 \le 2^{k+1}$, 
        갑자기 왜 이렇게 되는지 모르겠어"
```

(명료화 성공: 정확히 어디서 막혔는지 표현!)

---

### 명료화 질문

```
[MAICE] "좋은 질문입니다! 이 부분이 부등식의 연쇄성(chain inequality)을 
         사용하는 핵심 단계입니다.

혹시 다음 중 어떤 부분이 특히 헷갈리시나요?

1) (k+1)^2를 전개하는 과정?
2) 2^(k+1) = 2 × 2^k로 바꾸는 이유?
3) 귀납 가정(k^2 < 2^k)을 어떻게 사용하는지?
4) 부등식을 어떻게 이어가는지?

구체적으로 말씀해주시면 그 부분을 집중적으로 설명드리겠습니다."
```

---

### 학생 후속 질문 시리즈

```
[학생] "부등식의 연쇄성이 뭐야"

[MAICE] [연쇄성 개념 상세 설명]

[학생] "$2^k + 2k + 1 \le 2^{k+1}$ 을 확인하는 방법"

[MAICE] [단계별 검증 방법 설명]

[학생] "1번,2번,3번부분 표시해서 딱딱 알려줘"

[MAICE] [기저-귀납-결론을 명확히 번호로 구분하여 재구성]
```

---

### 분석

**QAC 평가**:
- 질문 점수: 2점 → 4점 (+2점 향상)
- 답변 점수: 5점
- 맥락 점수: 5점 (반복 명료화 22개 메시지)
- **총점**: 14점 / 15점

**학습 패턴**:
- 문제 제시 → 어려움 발견 → 구체적 질문 → 개념 이해
- 하위권 학생이 명료화를 통해 어려움을 극복하는 전형적 사례
- 최종적으로 완전한 이해에 도달

## 사례 4: 학습 맥락 확인형 (Context-Seeking Clarification)

**명료화 유형**: 맥락 파악형  
**학생 수준**: 중간고사 객관식 55.8점 (중위권)

```
[학생] "수학적 귀납법에 대해 알려줘"
```

**문제점**:
- 너무 포괄적
- 어떤 부분을 알고 싶은지 불명확

```
[MAICE] "수학적 귀납법의 어떤 부분이 특히 궁금하신가요?
```

## 사례 5: 명료화 요청형 (Clarification Request)

**세션 ID**: [익명]  
**명료화 유형**: 오류 대응형  
**학생 수준**: 중간고사 객관식 28.5점 (하위권)

### 무의미한 입력

```
[학생] "$xn\left\vert\larr\right\vert$$565_{weee_{\placeholder{}}}$ 1034234="
```

**문제점**:
- 수식 입력 오류
- 의미를 파악할 수 없음
- 학습 의도 불명

---

### 명료화 질문

```
[MAICE] "혹시 표현하신 식이나 기호를 조금 더 정확하게 써주실 수 있을까요?

어떤 수학 문제인지 제가 잘 이해할 수 있도록 다음을 알려주시면 좋겠습니다:

1) 어떤 문제를 풀고 계신가요?
2) 어떤 식을 표현하려고 하셨나요?
3) 지금 어떤 부분이 어려우신가요?

천천히 다시 질문해주세요 😊"
```

---

### 분석

**QAC 평가**:
- 질문 점수: 0점 (무의미)
- 답변 점수: 5점 (적절한 대응)
- 맥락 점수: 5점 (명료화 요청)
- **총점**: 10점 / 15점

**시스템 안정성**:
- 오류 입력에도 적절히 대응
- 학생을 비난하지 않고 재질문 유도
- 시스템 강건성(robustness) 입증

## 명료화 성공 기준

**성공 사례** (62건, 87%):
- 명료화 질문에 학생이 응답
- 응답을 통해 질문이 구체화됨
- 최종 답변이 제공됨

**부분 성공** (7건, 10%):
- 명료화 시도는 했으나, 학생 응답이 여전히 모호
- 2-3회 반복 명료화 필요

**실패 사례** (2건, 3%):
- 학생이 명료화 질문에 응답하지 않음
- 세션 중단

## 교육적 함의

1. **명료화 = 메타인지 촉진**: 학생이 "내가 무엇을 모르는지" 인식하게 함
2. **학생 주도 학습**: 명료화 후 능동적 질문으로 발전
3. **안전한 학습 환경**: 틀려도 괜찮다는 분위기 조성
4. **개별 맞춤**: 각 학생의 이해 수준에 맞는 지원

본 부록의 사례들은 명료화 프로세스가 단순한 기술적 기능을 넘어, 깊이 있는 학습을 유도하는 교육적 개입임을 보여준다.
