#### 7.3.1 Agent 강점 사례: C2(학습 지원) - 사고 과정 유도 (세션 75) ⭐

**학생 정보**: 
- 중간고사 총점: 40.8점 (하위권, Q1)
- 모드: Agent
- C2(학습 지원) 점수: 3/5 (사고 과정 유도=1)
- 총점: 31/40
- 메시지 수: 25개
- 세션 특징: 명료화를 통한 깊이 있는 학습 전개

**대화 전체 흐름**:

이 세션은 학생이 수학적 귀납법의 **원리에 대한 근본적인 의문**("왜 증명이 되는거야?")에서 시작하여, **실제 증명 문제**로 발전하는 전형적인 탐구형 학습 패턴을 보여준다.

**[1단계] 학생의 다중 질문 (메시지 0-2)**:

```
[학생 질문 1]: (k+1)! 을 다르게 나타낼 방법이 있어?

[학생 질문 2]: 수학적 귀납법으로 k, k+1일때 성립한걸 보인게 
              왜 증명이 되는거야?

[학생 질문 3]: 도미노 이론은 누가 만든거야?
```

**분석**: 학생이 세 가지 질문을 연속으로 제시하였다. 이는 수학적 귀납법에 대한 다양한 측면의 궁금증을 가지고 있음을 보여준다. 특히 질문 2 ("왜 증명이 되는거야?")는 귀납법의 **논리적 정당성**에 대한 깊은 의문으로, 단순 절차 암기를 넘어 원리를 이해하려는 태도를 나타낸다.

**[2단계] Agent의 명료화 (메시지 3-4)**:

```
[MAICE 분류]: 분류 결과를 바탕으로 명료화 질문을 선택하고 있습니다...

[MAICE 명료화]: "지금 말씀하신 '도미노 이론'은 수학적 귀납법에서 
                비유로 사용된 도미노 비유를 말하시는 걸까요, 
                아니면 실제로 누가 만든 이론을 알고 싶은 걸까요? 🤔"
```

**분석**: Agent는 학생의 세 질문 중 가장 명확하지 않은 "도미노 이론"에 대해 명료화를 시도한다. 이는 학생이 수학적 개념과 역사적 사실을 혼동할 수 있음을 파악하고, 정확한 학습 의도를 확인하려는 전략이다.

**[3단계] 원리 설명 (메시지 5-6)**:

```
[MAICE 원리 설명]: 
✨ 왜 수학적 귀납법으로 k와 k+1일 때 성립함을 보이면 증명이 될까요?

좋은 질문이에요! 수학적 귀납법은 모든 자연수에 대해 어떤 명제가 
참임을 증명할 때 사용하는 매우 강력한 방법이에요.

[도미노 비유 설명]
- 첫 번째 도미노가 쓰러진다 (n=1일 때 참)
- 한 도미노가 쓰러지면 다음 도미노도 쓰러진다 (n=k → n=k+1)
- 따라서 모든 도미노가 쓰러진다 (모든 n에 대해 참)

이것이 바로 수학적 귀납법의 핵심 원리입니다...
```

**분석**: Agent는 학생의 근본적인 의문("왜 증명이 되는가?")에 대해 **도미노 비유**를 사용하여 직관적으로 설명한다. 이는 추상적인 논리적 구조를 구체적인 물리적 현상으로 연결하여 이해를 돕는 효과적인 교수 전략이다.

**[4단계] 학생의 추가 탐구 (메시지 7)**:

```
[학생 재질문]: 수학적 귀납법 설명할때 사용되는 도미노 이론을 누가 만든거야?
```

**분석**: 학생이 도미노 비유에 흥미를 느끼고 그 역사적 배경을 추가로 질문한다. 이는 명료화와 원리 설명이 학생의 **지적 호기심을 자극**했음을 보여준다. 단순히 "이해했다"고 끝나는 것이 아니라, 더 깊이 알고 싶어하는 능동적 학습 태도가 나타났다.

**[5단계] 실제 증명 문제로 전환 (메시지 9~)**:

```
[학생 증명 요청]: 1+2+2^2+...+2^(n-1)=2^n-1 
                 이걸 수학적 귀납법으로 증명해줘

[MAICE 증명 제공]:
① 기초 단계 (n=1일 때)
  좌변: 1
  우변: 2^1 - 1 = 1
  ⇒ 성립

② 귀납 가정 (n=k일 때)
  1+2+2^2+...+2^(k-1) = 2^k-1 이라고 가정

③ 귀납 단계 (n=k+1일 때)
  1+2+...+2^(k-1)+2^k 
  = (2^k-1) + 2^k    [귀납 가정 사용]
  = 2·2^k - 1
  = 2^(k+1) - 1      [성립]

✅ 따라서 모든 자연수 n에 대해 성립함이 증명되었습니다.

... 이후 추가 질문과 심화 학습 계속 (총 25개 메시지)
```

**분석**: 학생이 원리를 이해한 후, 자연스럽게 **실제 증명 문제**를 요청한다. 이는 "이해 → 적용"의 이상적인 학습 과정이 명료화를 통해 촉진되었음을 보여준다.

**C2 평가 근거** (Gemini 1.5 Pro):

| 요소 | 값 | 근거 |
|------|-----|------|
| **C2-1. 사고 과정 유도** | **1** | "n! < 2^n이 언제까지 성립하는지 **직접 값을 대입하여 비교해 보는 연습**을 제안하며 학생 스스로 사고 과정을 거치도록 유도함" |
| C2-2. 이해도 확인 | 0 | "이해했나요?", "어려운 부분은 없었나요?" 등의 명시적 확인 질문 부재 |
| C2-3. 메타인지 촉진 | 1 | 학생이 자신의 학습 과정을 돌아보게 하는 질문 포함 |
| C2-4. 깊이 사고 유도 | 1 | "왜 이 방식이 정당한가?" 등의 깊이 있는 질문 제시 |

**교육적 가치**:

1. **원리 중심 학습**: 단순 절차 암기가 아닌, "왜?"라는 질문을 통해 논리적 구조 이해
2. **탐구적 태도 발달**: 도미노 비유 → 역사적 배경 → 실제 증명으로 자연스럽게 확장
3. **능동적 학습**: 25개 메시지를 통해 학생이 지속적으로 질문하고 탐구
4. **하위권 학생 지원**: 복잡한 개념을 구체적 비유로 설명하여 접근성 향상

**Freepass였다면**:
- 첫 질문에 즉시 증명 과정 제공
- 원리에 대한 의문("왜?")이 해소되지 않을 수 있음
- 도미노 비유 같은 직관적 설명 없이 형식적 증명만 제공
- 학생의 지적 호기심이 자극되지 않아 추가 탐구 발생하지 않을 가능성

**정량적 효과**:
- 메시지 수: 25개 (명료화 없는 세션 평균 4.9개의 5배)
- 학습 시간: 약 40분 추정 (일반 세션의 5배)
- 학습 깊이: 원리 이해 + 실제 적용 + 심화 탐구

**분석**: 학생이 "정의"를 선택하자, Agent는 **정의 중심**의 설명을 제공한다. Freepass였다면 정의와 예시를 모두 포함한 긴 설명을 제공했을 것이나, Agent는 학생의 명확한 요구에 맞춰 정의에 집중한다.

**[6단계] 2차 명료화 및 심화 (메시지 8~)**:

#### 7.3.5 실패 사례 1: 학생의 불만과 이해도 부족 (세션 74, Freepass) ❌

```
[학생 질문 1]: 1^2 + 2^2 + 3^2 + ... + n^2 = n(n+1)(2n+1)/6 
              이 성립함을 수학적 귀납법으로 증명해줘

[MAICE 즉시 증명]:
좋습니다. 주어진 식 ... 을 수학적 귀납법으로 증명해보겠습니다.

① 귀납법의 기초 단계 (n=1일 때)
   좌변: 1^2 = 1
   우변: 1(1+1)(2·1+1)/6 = 1·2·3/6 = 1
   ⇒ 좌변 = 우변, 성립

② 귀납 가정
   어떤 자연수 k에 대하여 ... 이 성립한다고 가정

③ 귀납 단계 (n=k+1일 때)
   ... (상세한 대수 전개)
   
✅ 결론: 모든 자연수 n에 대해 성립함이 증명되었습니다.
```

**분석**: Freepass는 명료화 없이 즉시 **완전한 증명 과정**을 제공한다. 이 시점에서는 학생이 증명의 어느 부분을 어려워하는지, 어떤 수준의 설명이 필요한지 전혀 파악되지 않았다.

**[2단계] 학생의 두 번째 증명 요청 (메시지 2-3)**:

```
[학생 질문 2]: 그럼 1*2 + 2*3 + 3*4 + ... + n(n+1) = n(n+1)(n+2)/3 
              이 성립함을 수학적 귀납법으로 증명해줘

[MAICE 또 다른 완전 증명]:
좋아요. 이번에는 제시된 등식을 수학적 귀납법으로 증명해보겠습니다.

① 귀납의 기초 단계 (n=1)
   좌변: 1·2 = 2
   우변: 1(1+1)(1+2)/3 = 2
   ⇒ 성립

② 귀납 가정
   ... 

③ 귀납 단계 (n=k+1)
   ... (복잡한 대수 전개)
   
   이제 (k+3) = ((k+1)+2)이므로,
   ∴ (k+1)((k+1)+1)((k+1)+2)/3
   
✅ 따라서 n=k+1일 때도 동일한 형태의 식이 성립합니다.
```

**분석**: MAICE는 또 다시 **완전하고 형식적인 증명**을 제공한다. 특히 마지막 부분의 "(k+3) = ((k+1)+2)"와 같은 **형식적 변환**이 포함되어 있는데, 이것이 학생에게 혼란을 줄 것이다.

**[3단계] 학생의 이해 부족 표출 (메시지 4-5)**:

```
[학생 질문 3]: 이제 (k+3)=((k+1)+2)(k+3)=((k+1)+2)이므로
              이 부분이 이해가 안돼

[학생 불만 폭발]:
**아니 왜? 대체 왜 이렇게 복잡하게 하는건지 모르겠네**
**그냥 어차피 (k+1)(k+2)(k+3)/3 인거잖아**
**왜 그렇게 장황하게 풀어서 성립하는지 봐야하지?**
```

**분석**: 학생이 **강한 불만**을 표출한다. 이 불만은 여러 층위의 문제를 드러낸다:

1. **형식적 변환의 이해 부족**: "(k+3) = ((k+1)+2)" 같은 변환이 왜 필요한지 이해 못함
2. **수학적 귀납법의 본질 오해**: "어차피 같은 식"이면 왜 형태를 바꿔야 하는지 의문
3. **학습 좌절감**: "복잡하게", "장황하게" - AI의 설명 방식에 대한 불신
4. **이해도-설명 수준 불일치**: MAICE의 형식적 설명이 학생 수준을 벗어남

**[4단계] MAICE의 재설명 시도 (메시지 6-7)**:

```
[MAICE 재설명 1]:
정확히 짚으셨어요 👏  
사실 본질적으로 말씀하신 게 **완전히 맞습니다.**

실제로 (k+1)(k+2)(k+3)/3까지만 나와도 이미 증명 완료예요.
(k+3) = ((k+1)+2) 같은 변형은 단지 "형태적으로 맞춘다"는 의미일 뿐...

[MAICE 재설명 2]:
좋은 질문이에요 😊  
... (더 자세한 설명)
```

**분석**: MAICE는 학생의 불만을 인지하고 재설명을 시도하지만, **이미 학생이 좌절한 후**이다. Freepass의 근본적 한계가 드러나는 순간이다.

**문제 진단**:

| 문제 유형 | 세부 내용 | 원인 |
|---------|----------|------|
| **이해도 불일치** | 형식적 증명 vs 학생의 직관적 이해 | 사전 파악 불가 |
| **설명 수준 과다** | 대학 수준 엄밀성 vs 고등학생 필요 | 맞춤화 부족 |
| **타이밍 문제** | 불만 표출 후 재설명 | 사전 예방 불가 |
| **동기 저하** | "복잡하다", "장황하다" | 학습 의욕 감소 |

**Agent였다면**:

```
[1단계] 명료화:
   "증명 과정에서 어떤 부분이 궁금하신가요? 
    형식적인 증명 전 과정인가요, 아니면 특정 단계만 알고 싶으신가요?"
    
[2단계] 학생 응답 파악:
   → 만약 "간단하게"라고 답하면 핵심만 설명
   → "형식적으로"라고 답하면 엄밀한 증명 제공
    
[3단계] 이해도 확인:
   "귀납 단계에서 식 변형이 이해되시나요?"
   → 이해 부족 감지 시 즉시 재설명
    
[4단계] 사전 예방:
   "형식적 변환은 생략 가능합니다. 필요하신가요?"
   → 학생이 불필요하다고 하면 생략
```

**교육적 시사점**:

1. **Freepass의 근본적 한계**: 
   - 학생 이해도를 사전에 파악할 수 없음
   - 설명 수준을 맞추기 어려움
   - 불만이 표출된 후에야 문제 인지

2. **하위권 학생에게 특히 문제**:
   - 형식적 증명이 과도하게 복잡하게 느껴짐
   - 본질과 형식을 구분하지 못함
   - 좌절감으로 학습 동기 저하

3. **개선 방향**:
   - Freepass에도 간단한 수준 확인 질문 추가 고려
   - "간단한 증명" vs "엄밀한 증명" 옵션 제공
   - 학생의 감정적 신호("복잡하다") 감지 및 대응

#### 7.3.7 실패 사례 3: 명료화 거부 (세션 116, Agent) ❌

**학생 정보**: 
- 모드: Agent
- 총점: 8/40 (최저 점수대, 공동 최하위)
- 메시지 수: 3개 (매우 짧음)
- 문제: 학생이 명료화 프로세스를 거부하고 즉시 답변 요구

**대화 전체 흐름**:

이 세션은 Agent 모드의 효과가 **학생의 협조를 전제**로 한다는 중요한 한계를 보여준다. 학생이 명료화를 거부하면 Agent는 Freepass처럼 작동하게 되며, C2(학습 지원)의 효과를 발휘할 수 없다.

**[전체 대화]**:

```
[학생 메시지 0]: (이미지 업로드)
                이미지를 수식으로 변환 중입니다...
                문제 2: n ≥ 4인 모든 자연수 n에 대하여 
                부등식 n! > 2^n 이 성립함을 수학적 귀납법으로 증명하시오.

[학생 메시지 1]: **내가 물어본 저 문제를 풀어줘**

[MAICE 응답]:
✨ 수학적 귀납법으로 n! > 2^n 임을 증명하기

안녕하세요 😊  
이번 문제는 수학적 귀납법을 사용하여, n ≥ 4인 모든 자연수 n에 대해

```

**문제 분석**:

1. **명료화 프로세스 미작동**: 
   - 학생이 "내가 물어본 저 문제를 풀어줘"라고 **강하게 요구**
   - Agent가 명료화를 시도할 시간적 여유 없음
   - 학생이 이미 문제를 명확히 제시했으므로 명료화 필요성 낮다고 판단 가능

2. **즉시 답변 제공**:
   - Agent가 Freepass처럼 작동
   - 명료화 없이 즉시 증명 과정 제공
   - C2(학습 지원) 기능 전혀 발휘 안됨

3. **짧은 대화로 종료**:
   - 3개 메시지로 종료 (Agent 평균 20개의 15%)
   - 추가 탐구, 이해도 확인, 심화 학습 전무

**전체 점수 8/40의 의미**:

| 영역 | 점수 | Agent의 기대 효과 | 실제 결과 |
|------|------|-----------------|----------|

| B (답변) | 2/15 | 맞춤형 설명 | 일괄 설명 |
| **C (맥락)** | **1/10** | **학습 지원 (C2)** | **전혀 작동 안함** |

특히 C2(학습 지원) 점수가 1/5로 최저인 것은 Agent의 핵심 기능이 전혀 발휘되지 않았음을 의미한다.

**Agent가 실패한 이유**:

```

  1. 질문 수신
  2. 명료화 필요성 판단
  3. 명료화 질문 제공
  4. 학생 응답 수신
  5. 맞춤형 답변 제공

  1. 질문 수신 (문제 제시)
  2. 학생의 강한 요구 ("풀어줘")
  3. ❌ 명료화 스킵
  4. 즉시 답변 제공
  5. 짧은 대화로 종료

```

**학생의 심리**:

학생이 "내가 물어본 저 문제를 풀어줘"라고 표현한 것은:
- 문제가 이미 명확하다고 생각함
- 시간이 부족하거나 급함
- 명료화 질문을 기다리고 싶지 않음
- 즉시 답변을 강하게 원함

이는 일부 학생에게 명료화 프로세스가 **번거롭거나 불필요**하게 느껴질 수 있음을 보여준다.

**교육적 시사점**:

1. **Agent 모드의 전제 조건**:
   - 학생이 명료화 프로세스의 가치를 이해해야 함
   - 시스템 사용 전 **오리엔테이션 필수**
   - "왜 명료화를 하는가"에 대한 사전 교육

2. **학생 선호도의 중요성**:
   - 일부 학생은 즉시 답변을 선호
   - 시간 제약이 있는 상황(시험 전 등)에서는 Freepass가 더 적합
   - **학생이 모드를 선택**할 수 있게 하는 것이 중요

3. **Agent 개선 방향**:
   - 명료화 스킵 옵션 제공: "바로 답변을 원하시면 ✓"
   - 학생의 급한 신호("풀어줘", "빨리") 감지
   - 상황에 따라 명료화 강도 조절

4. **실험 설계의 한계**:
   - 본 연구는 학생을 무작위로 Agent/Freepass에 배정
   - 학생의 선호도를 고려하지 않음
   - 선호도와 효과의 상호작용 분석 필요

### 7.6 다중 평가자 분석: QAC 체크리스트 검증

#### 7.6.1 평가 도구 개선 및 공정성 확보

평가자 간 신뢰도를 검증하기 위해 3개의 독립적인 AI 모델(GPT-4o, Claude 3.5 Sonnet, Gemini 2.5 Flash)을 사용하여 QAC 체크리스트 루브릭의 신뢰성을 확인하였다.

**QAC 체크리스트 루브릭 특징**:
- **40점 만점, 8개 항목** (A1~A3, B1~B3, C1~C2)
- **32개 체크리스트 요소** (항목당 4개 요소, 0/1 판단)
- **객관적 평가**: 각 요소별 충족 여부 + 근거 문장 제시
- **자동 점수 산정**: 충족 개수 + 1 (0개=1점, 4개=5점)

**평가 항목 구조**:

**A. 질문 영역** (15점):
- A1. 수학적 전문성 (5점): 개념 정확성, 용어 사용, 논리적 구조, 표현 명확성
- A2. 질문 구조화 (5점): 핵심 질문 명확성, 조건 제시, 구조 논리성, 의도 명시
- A3. 학습 맥락 (5점): 학습 단계, 선수학습, 어려움 명시, 학습 목표

**B. 답변 영역** (15점):
- B1. 학습자 맞춤도 (5점): 수준 적합성, 선수지식 고려, 난이도 조절, 학습 스타일
- B2. 설명 체계성 (5점): 논리적 흐름, 단계별 설명, 구조적 정리, 핵심 강조
- B3. 학습 확장성 (5점): 심화 방향, 응용 연계, 오개념 교정, 자기주도 유도

**C. 맥락 영역** (10점):
- C1. 대화 일관성 (5점): 맥락 유지, 답변 연결, 목적 일치, 정보 통합
- C2. 학습 지원 (5점): 사고 과정 유도, 이해도 확인, 메타인지 촉진, 깊이 있는 사고

**다중 평가자**:
- Gemini 2.5 Flash (Google) - 284개 세션
- Claude 4.5 Haiku (Anthropic) - 221개 세션
- GPT-5 mini (OpenAI) - 284개 세션

**참고**: Claude는 221개 세션만 분석 (일부 세션 처리 오류), Gemini와 GPT-5는 전체 284개 분석

**구체적 사례 분석**:

Freepass 학생의 어려움 명시 사례:
- "기하급수 합 공식 안배웠음" (세션 212) - 선수학습 부족
- "표준형으로 바꾸는 거 공식은 알겠는데 그래프는..." (세션 100) - 특정 개념 어려움
- "k+1 단계에서 식 전개가 막혀요" (다수 세션) - 절차적 어려움

Agent 학생의 특징:
- 어려움을 명시하기보다 일반적 질문 ("알려줘", "뭐야")
- Agent가 명료화로 어려움을 파악하므로 굳이 명시할 필요 없음
- 명료화에 의존하는 패턴 형성

**메커니즘**:

```
Freepass 환경:
  → 명료화 없음
  → 학생이 충분한 정보 제공하지 않으면 적절한 답변 못 받음
  → "어려움을 명시해야 원하는 답을 받는다"는 학습
  → 메타인지적 자기표현 능력 발달
  
Agent 환경:
  → 명료화 있음
  → Agent가 질문을 통해 어려움 파악
  → 학생은 질문에 답하기만 하면 됨
  → 스스로 표현할 필요성 감소
  → 명료화 의존 패턴
```

**교육적 함의**:

**Freepass의 장점**:
1. **메타인지 발달**: 자신의 학습 상태를 스스로 분석·표현
2. **자기주도성 향상**: 어려움을 능동적으로 명시
3. **의사소통 능력**: 구체적이고 명확한 표현 연습

**Freepass의 단점**:
1. **하위권 부담**: 자기 분석 능력이 부족한 학생에게는 과도한 요구
2. **인지 부하**: 학습 내용 + 자기 분석 동시 요구
3. **불확실성**: 충분한 정보를 제공했는지 확신 어려움

**적용 권장**:
- **자기주도 학습 능력 있는 학생**: Freepass로 메타인지 발달 촉진
- **하위권 또는 메타인지 부족 학생**: Agent로 체계적 지원
- **학습 단계별**: 초기에는 Agent → 점차 Freepass로 전환 (비계 제거)

**C2 (학습 지원) - Agent 우세 (2개 모델 일치)**:

**C2-1 (사고 과정 유도) - Agent 우수 사례**:

- "**수학적 귀납법의 어느 부분을 더 자세히 알고 싶으신가요? 😊 예를 들면, 논리적인 이유나 예시 풀이 중에서요.**" (세션 61, 메시지 8)
- "**어떤 부분을 더 자세히 알고 싶으신가요? 예를 들어 논리 구조나 증명 과정 중 어느 부분인지 알려주시면 좋겠어요**" (세션 61, 메시지 13)
- 명료화 자체가 이해도 확인의 역할: "**정의를 알고 싶으신가요, 아니면 어떻게 사용하는지 예시로 설명해드릴까요?**" (세션 61, 메시지 3)

**메커니즘**:

```
Agent의 명료화 = 사고 유도 + 이해도 확인

예시: "정의를 알고 싶으신가요, 예시를 알고 싶으신가요?"
  ↓
학생 사고 과정:
  1. "내가 지금 무엇을 모르는가?" (자기 분석)
  2. "정의가 필요한가? 예시가 필요한가?" (메타인지)
  3. "정의요" (명확한 선택)
  ↓
Agent의 이해도 파악:
  - 학생이 개념적 이해를 원함 (절차 아님)
  - 정의 중심 설명 제공
  ↓
추가 명료화:
  - "어느 부분을 더 알고 싶으신가요?"
  - 학생의 이해도 지속 확인
```

**Freepass와의 차이**:

| 측면 | Agent (C2 우수) | Freepass |
|------|----------------|----------|
| 사고 유도 | 명료화 질문으로 사고 촉진 | 답변만 제공, 사고 유도 없음 |
| 이해도 확인 | 명료화 = 이해도 점검 | 이해도 파악 어려움 |
| 상호작용 | 20개 메시지, 깊은 대화 | 4개 메시지, 짧은 대화 |
| 학생 역할 | 사고하고 선택 | 수동적 수용 |

**교육적 함의**:

**Agent의 장점**:
1. **사고 명시화**: 명료화 질문이 학생의 사고를 드러나게 함
2. **이해도 실시간 파악**: 학생 응답으로 이해 수준 확인
3. **맞춤형 지원**: 이해도에 따라 설명 수준 조절
4. **깊이 있는 학습**: 반복적 확인으로 깊이 있는 이해 유도

**Agent가 특히 효과적인 상황**:
1. 하위권 학생 (이해도 확인 필수)
2. 복잡한 개념 학습 (단계별 확인 필요)
3. 오개념 교정 (사고 과정 파악 필요)
4. 문제 해결 학습 (사고 과정 유도 중요)

#### 7.6.5 하위권 학생 효과: C2 항목에 국한 ⭐⭐⭐

본 연구의 가장 중요한 발견 중 하나는 **Agent의 효과가 학업 하위권 학생에게 특히 강하게 나타난다**는 것이다. 이는 C2(학습 지원) 항목에 집중되어 있으며, 2개 독립 모델에서 재현되었다.

**하위권 학생 정의**:
- 중간고사 총점 ≤42점 (하위 25%, Q1)
- Agent: 24개 세션
- Freepass: 35개 세션

[표 7-4] Q1 하위권(중간고사 ≤42점) C2 항목 비교

| 모델 | Agent M (SD) | Free M (SD) | 차이 | t | p-value | Cohen's d | 효과 크기 |
|------|--------------|-------------|------|---|---------|-----------|----------|
| Gemini | 1.80 (1.06) | 1.59 (0.85) | +0.21 | 1.72 | 0.087 | 0.231 | 작음 (경향) |
| **Claude** | **2.83 (1.55)** | **2.16 (1.43)** | **+0.68** | **2.83** | **0.005*** | **0.459** | **중간** |

- Claude와 GPT-5 모델에서 하위권 학생에 대한 Agent의 C2(학습 지원) 효과가 **통계적으로 매우 유의하게** 발견 (p<0.01)
- 효과 크기는 **중간 수준** (d = 0.41 ~ 0.46)
- 2/3 모델 재현으로 **신뢰성 확보**

**중상위권 학생과의 대조 (C2 항목)**:

| 학생 그룹 | Agent M | Free M | 차이 | p-value |
|---------|---------|--------|------|---------|
| **Q1 (하위권)** | **2.38** | **1.83** | **+0.55** | **0.005** ✅ |
| Q2 (중하위권) | 2.45 | 2.31 | +0.14 | 0.412 |

**패턴**: 
- 하위권(Q1)에서만 통계적으로 유의한 차이 (p<0.01)
- 중상위권(Q2-Q4)에서는 차이 없음 (모든 p>0.39)
- 이는 Agent의 C2 효과가 **하위권 학생에게 특화**되어 있음을 보여줌

**하위권 학생 C2 세부 요소 분석**:

| 요소 | Agent 충족률 | Free 충족률 | 차이 | p-value | OR (Odds Ratio) |
|------|-------------|-------------|------|---------|-----------------|
| **C2-1. 사고 과정 유도** | **41%** | **22%** | **+19%p** | **0.009*** | **2.47** |
| **C2-2. 이해도 확인** | **46%** | **28%** | **+18%p** | **0.012*** | **2.18** |
| C2-3. 메타인지 촉진 | 24% | 18% | +6%p | 0.325 | 1.42 |
| C2-4. 깊이 사고 유도 | 32% | 21% | +11%p | 0.089 | 1.74 |

**해석**: 
- 하위권 Agent 학생은 **사고 과정 유도**를 받을 확률이 Freepass보다 **2.47배** 높음 (OR=2.47)
- **이해도 확인**을 받을 확률도 **2.18배** 높음 (OR=2.18)
- 이 두 요소가 하위권 학생의 학습 효과를 크게 향상시키는 핵심 메커니즘

**왜 하위권 학생에게만 효과가 강한가?**:

```
하위권 학생의 특징:
  - 개념 이해 부족
  - 자기 점검 능력 미흡
  - 어디서부터 모르는지 모름
  - 질문 구성 능력 부족
  
Agent의 C2 지원이 필수적인 이유:
  ↓
1. 사고 과정 유도 (+19%p):
   → "어떤 부분이 어려운가요?" 
   → 학생이 자신의 어려움을 구체화하도록 유도
   → 막연한 "모름" → 구체적 "이 부분을 모름"
   
2. 이해도 확인 (+18%p):
   → "k+1 단계가 이해되시나요?"
   → 학생의 현재 이해 수준을 실시간 파악
   → 이해 부족 감지 시 즉시 재설명
   
3. 단계별 안내:
   → "먼저 n=1부터 확인해볼까요?"
   → 작은 단계로 분해
   → 성공 경험 누적
```

**중상위권 학생은 왜 차이가 없는가?**:

```
중상위권 학생의 특징:
  - 기본 개념 이미 이해
  - 자기 점검 능력 있음
  - 스스로 질문 구성 가능
  
Agent의 C2 지원:
  ↓
효과 제한적:
  - 이미 사고 과정 유도 없이도 스스로 사고
  - 이해도 확인 없이도 스스로 점검
  - 명료화가 오히려 시간 낭비로 느껴질 수 있음
  
Freepass도 충분:
  - 즉시 답변 받아서 스스로 학습
  - 자기주도적으로 추가 질문
  - A3(맥락 명시)에서 오히려 우수
```

**실제 사례 비교**:

**테마 2: Rubber Duck 효과 - 자기 설명을 통한 이해** ⭐⭐⭐

Agent 모드 학생 [S06]:
> "AI에 **프롬포트를 입력하는 도중에 갑자기 이해**가 되기 시작했다. **내가 AI에게 나의 상황을 설명하다 보니**, 자연스럽게 **해결 방안이 떠올랐다**"

**교육심리학적 의미**:
- **Self-Explanation Effect** (Chi et al., 1994)
- Agent의 명료화 요구 → 학생의 자기 설명 → 스스로 이해 발생
- AI가 직접 답을 주지 않아도 학습 효과 발생
- 프로그래밍의 "Rubber Duck Debugging"과 동일한 메커니즘

**통계** (학생 ID는 익명화, 부록 F 참조):
- **Freepass 사용자**: 4/8명 (50.0%) B 방식 선호 [S31, S39, S51, S53], 4/8명 (50.0%) A 방식 선호 [S34, S37, S40, S54]
- **Agent 사용자**: 7/9명 (77.8%) B 방식 선호 [S04, S05, S06, S09, S15, S20, S28], 2/9명 (22.2%) A 방식 선호 [S03, S08]

**주목할 점**: 
- Agent 그룹의 B 선호율(77.8%)이 Freepass 그룹(50.0%)보다 높음
- **하지만** Agent의 A 선호자 2명(S03, S08)이 "B의 교육적 가치는 인정하지만 A 선택"

**Freepass 사용자의 B 방식 선호 이유**:

학생 [S39]:
> "결론적으로 저는 **B 방식**, 즉 AI가 질문을 통해 생각하게 만든 다음 답을 주는 방식을 선호합니다. [...] 저는 AI와의 대화가 단순히 문제 해결을 넘어 **학습과 성장의 과정이 되기를 바랍니다**"

학생 [S53]:
> "저는 **B 방식**을 더 선호합니다. AI가 질문을 던져 **스스로 생각하게 한 뒤** 답을 확인하면 이해가 깊어지고 문제 해결 능력이 향상되기 때문입니다"

**Agent 사용자의 B 방식 선호 이유**:

학생 [S06] (신규):
> "**B**. A방식이 바로바로 답을 알 수 있어서 편하고 쉽겠지만, B방식처럼 진행한다면 **공부 과정에 나의 생각이 더 들어갔기 때문에** 더욱 **오래, 깊게 공부 내용이 남을 것**이다"

학생 [S04]:
> "B 방식은 내가 직접 생각을 해보게되서 A 방식보다 **기억에 더 오래 남는다**"

**Agent 사용자의 A 방식 선호 이유 (Value-Convenience Gap)** ⭐:

Agent를 경험했지만 A 방식을 선호한 학생들(S03, S08; 2명, 22.2%)은 **교육적 가치는 인정하지만 편의성을 우선**하는 특징을 보였다.

학생 S28 (Agent_B, B 선호지만 불만 표현):
> "B 방식이 저희의 **발전에 더 도움이 될것 같지만** 만약 사용한다면 **A를 쓸거 같아요**.... AI를 사용할때 저는 **빠른 답변을 요구**하는데 B방식은 **점점 길어질거 같아요**..."

학생 S09 (Agent_B, 하지만 UX 불만):
> "B방식이 **학습에는 더 도움이 될 것 같긴한데** **채팅 치는게 너무 귀찮다**"

**실제 대화 패턴 분석**:
- Agent_A 그룹: 세션당 평균 1.5개 메시지 (Agent_B 2.2개보다 낮음)
- 평균 메시지 길이: 68자 (Agent_B 130자보다 짧음)
- 대화 중 불만 표현: "주절주절 뱅글뱅글", "귀납법 설명 자제"

**Value-Convenience Gap (가치-편의성 괴리)**:
```
인지: "B가 발전/학습에 도움" (교육적 가치 인정)
  ↓
의도: "하지만 A를 쓸 것" (실제 선택)
  ↓
이유: "빠른 답변", "채팅 귀찮다", "과정이 길다"
```

이는 **기술 수용 모델 (TAM)**에서 말하는 "인식된 유용성"보다 "인식된 사용 편의성"이 실제 사용 의도에 더 큰 영향을 미침을 보여준다 (Davis, 1989). 또한 **인지 부하 이론 (Cognitive Load Theory)**관점에서, 명료화 프로세스의 외재적 부하(질문 작성, 대화 유지)가 일부 학생에게는 과도했음을 시사한다.

**교육적 함의 (블라인드 실험의 이중 증거)**:

1. **Freepass → B 선호 (50.0%)**: 경험하지 않은 학생들의 절반이 교육적 가치 인정
   - 블라인드 실험의 객관성 증명
   - Agent 접근의 교육적 타당성 확인

2. **Agent → A 선호 (22.2%)**: 경험한 학생 중 일부가 편의성 우선
   - **UX 개선 필요성** 제기
   - 교육적 가치 ≠ 사용 편의성
   - 시스템 설계의 실천적 과제

**통합 해석**:
- Agent의 교육적 가치는 **경험자(77.8%)와 비경험자(50.0%) 모두 인정**
- Agent 경험자의 B 선호율이 더 높음 (경험 → 가치 인식 ⬆️)
- 하지만 일부 경험자는 **UX 장벽으로 인해 사용을 꺼림**
- → 명료화 프로세스 자체의 문제가 아닌, **인터페이스와 진행 방식 개선 필요**

**설문 점수 분석: Freepass 그룹 내 양극화** ⭐⭐⭐

설문의 리커트 척도 분석 결과, Freepass 그룹 내에서 B 방식 선호자와 A 방식 선호자 간에 큰 차이가 발견되었다:

| 항목 | Free_B | Free_A | 차이 |
|------|--------|--------|------|
| **AI 이해도** | 4.75 | 3.50 | +1.25 ⭐ |
| **맥락 제공** | 4.75 | 3.50 | +1.25 ⭐ |
| **질문 생성** | 4.25 | 3.00 | +1.25 ⭐ |
| **증명 자기검증** | 4.50 | 3.25 | +1.25 ⭐ |
| **메타인지: 자기평가** | 5.00 | 4.00 | +1.00 |

**해석**: 
- Freepass_B (B 선호자)는 실제로 **더 높은 메타인지와 질문 능력**을 보유
- → B 방식 선호는 우연이 아닌, **자신에게 필요한 것을 정확히 인식**
- → 능동적 학습자는 Freepass의 즉시 답변으로는 만족하지 못함
- → **명료화 프로세스에 대한 잠재적 수요** 존재

**Agent_A의 역설**:

역설적으로 Agent_A 그룹(Agent 경험 + A 선호)은 만족도에서 만점(5.00)을 보였으나, 실제 대화 참여도는 가장 낮았다(세션당 1.5개 메시지). 이는 **리커트 척도만으로는 실제 사용 의도와 행동을 파악하기 어려움**을 보여준다. 서술형 응답("채팅 치는 게 귀찮다")에서만 Value-Convenience Gap이 드러났다.

#### 7.7.3 양적-질적 통합 해석

| 분석 유형 | 주요 발견 | 의미 |
|----------|----------|------|
| **양적 (QAC)** | 총점 차이 없음 (p>0.50) | 두 모드의 전반적 효과 유사 |
| **양적 (QAC)** | A3 Free 우세, C2 Agent 우세 | 차별적 강점 존재 |
| **양적 (설문)** | 만족도 차이 없음 (p>0.80) | 주관적 만족도도 유사 |
| **양적 (설문)** | Free_B vs Free_A 차이 1.25점 | Freepass 내 능력 양극화 ⭐ |
| **질적 (서술형)** | Agent "원리", Free "힌트" | 다른 학습 경로 |
| **질적 (블라인드)** | Free 50% vs Agent 77.8% B 선호 | 경험 → 가치 인식 강화 |
| **질적 (참여도)** | Agent_A 1.5개, Free_B 2.7개 메시지 | 선호도와 실제 행동 일치 |

**통합 해석**:
- 두 모드가 **다른 경로로 유사한 학습 결과** 도달
- Agent: Dewey(1910)의 "문제 명료화" 과정을 통한 구조적 이해
- Freepass: 직접 지원을 통한 효율적 문제 해결
- 블라인드 실험 증거: 경험하지 않은 학생들도 질문 기반 접근의 우수성 인정

**삼각검증의 가치**:

1. **양적 (QAC + 설문)**: "무엇이" 다른지 측정 → 총점 차이 없음, 세부 차이 있음
2. **질적 (서술형)**: "왜", "어떻게" 다른지 규명 → 학습 경로, 메커니즘, 감정
3. **행동 (참여도)**: 실제 행동 확인 → 선호도와 참여도 일치, Value-Convenience Gap

**3가지 통찰**:

1. **Free_B의 능력**: 리커트 점수에서 Free_A보다 1.25점 높음
   - → B 선호는 우연이 아닌, 높은 학습 역량의 결과

2. **Agent_A의 역설**: 만족도 5.00, 참여도 1.5개 메시지
   - → 리커트 척도의 한계, 서술형에서만 "채팅 귀찮다" 드러남

3. **경험 vs 직관**: Agent 77.8% vs Freepass 50.0% B 선호
   - → 경험이 교육적 가치 인식 강화

- 우수 명료화 비율: 66.1% (명료화 수행 98개 중 78개)
- 명료화 있는 세션의 메시지 수: 2.4배 증가

**2. 총점에서는 차이 없음**:
- 3개 모델 모두 Agent와 Freepass 간 총점에서 유의미한 차이 없음 (모든 p>0.50)
- 평가 도구의 공정성 확보 (C1 편향 제거)

**3. 차별적 강점 발견**:

**A3 (학습 맥락) - Freepass 우세** (3개 모델 모두 p<0.01):
- Free는 처음부터 충분한 맥락 정보를 제공해야 하므로 메타인지적 표현 능력 발달
- 특히 구체적 어려움 명시에서 18%p 우세 (p=0.004)

**C2 (학습 지원) - Agent 우세** (2개 모델 p<0.05):
- Agent는 사고 과정 유도(+12%p, p=0.027)와 이해도 확인(+11%p, p=0.041)에서 우세
- 명료화 대화를 통한 체계적 학습 지원

**4. 하위권 학생에 대한 특수 효과** (p<0.01, d~0.4):
- Claude와 GPT-5에서 하위권 학생의 C2 항목에서 Agent 명확한 우세
- 사고 과정 유도(+19%p)와 이해도 확인(+18%p)에서 특히 효과적
- 하위권 학생에게 Agent의 체계적 지원이 매우 중요함을 입증

**5. 평가자 간 신뢰도 확보**:
- ICC=0.595 (보통), Cronbach's α=0.840 (좋음)
- 3개 독립 모델의 일관된 결과로 재현성 확보

**6. 학생 설문 분석 - 삼각검증** (n=34, 58.6% 응답률):
- **Agent 강점**: 문제해결 능력(+0.23점), 증명 검증(+0.17점)
- **Freepass 강점**: AI 이해도(-0.34점), 메타인지(-0.33점), 질문 명확성(-0.22점)
- **전체 만족도**: 거의 동등 (Agent 4.17점 vs Freepass 4.22점, 차이 -0.05점)
- **수렴적 증거**: QAC 채점, 설문, 서술형 모두 **두 모드의 차별적 강점** 일관되게 지지
- **블라인드 실험**: 학생들은 모드를 모르는 상태에서 응답하여 편향 통제

#### 7.8.2 이론적 함의

**1. 차별적 강점의 발견**:
- Agent와 Freepass는 각기 다른 교육적 강점을 가짐
- Agent: 체계적 학습 지원 (특히 하위권)
- Freepass: 메타인지적 자기표현 능력

**2. 총점 vs 세부 분석의 중요성**:
- 총점 비교만으로는 교육적 효과의 실체를 파악하기 어려움
- 세밀한 항목별 분석을 통해 차별적 효과 규명 가능

**3. 학습자 특성에 따른 맞춤형 접근의 필요성**:
- 하위권 학생: Agent의 체계적 지원이 효과적
- 상위권 학생: 자율성과 구조화 모두 가능
- 학습자 수준에 따른 차별화된 교수 전략 필요

#### 7.8.3 실천적 시사점

**1. 적용 권장사항**:

**하위권 학생** (중간고사 하위 25%):
- Agent 모드 우선 권장 (통계적 근거: p<0.01, d~0.4)
- 사고 과정 유도와 이해도 확인을 통한 체계적 지원
- 작은 단계로 분해하여 성공 경험 제공

**중상위권 학생**:
- 혼합 전략 또는 학생 선호도 반영
- Agent로 시작하여 점진적으로 자율성 확대
- 필요 시 Agent로 재구조화

**2. 교수 전략**:

**초기 세션 (1-2회)**:
- Agent로 질문 구성 방법 학습
- 명료화 프로세스 경험

**중기 세션 (3-4회)**:
- 점진적 자율성 확대
- 학생 선호도 및 학습 패턴 관찰

**필요 시**:
- Agent로 재구조화
- 개념 이해 부족 시 체계적 지원

**3. 시스템 개선 방향**:

**Agent 강화**:
- C2 (학습 지원) 기능 강화
- 사고 과정 유도 및 이해도 확인 자동화
- 하위권 학생 탐지 및 맞춤 지원

**Freepass 보완**:
- A3 (학습 맥락) 정보 활용 강화
- 학생의 자기표현을 학습 데이터로 활용
- 메타인지 발달 지원 기능 추가

**통합 시스템**:
- 학생 수준 자동 판별
- 모드 간 자동 전환 기능
- 개인별 학습 이력 기반 맞춤형 제공

#### 7.8.4 연구의 한계 및 후속 연구

**제한점**:

### 7.6 교사 평가와 AI 모델 채점 비교

교사의 직접 평가와 3개 AI 모델(OpenAI GPT-5 mini, Gemini, Anthropic Haiku 4.5)의 자동 채점 결과를 비교하여 AI 채점의 타당성과 한계를 분석하였다.

#### 7.6.1 AI 모델별 교사 평가와의 상관관계

54개의 교사 평가 세션을 3개 AI 모델로 채점한 결과:

| 모델 | 전체 점수 상관계수 | MAE | RMSE | 분석 샘플 수 |
|------|------------------|-----|------|------------|
| **Anthropic Haiku 4.5** | **0.631*** | 5.26 | 6.71 | 50 |
| **OpenAI GPT-5 mini** | 0.591*** | **5.06** | **6.31** | 52 |
| Gemini | 0.351* | 7.12 | 8.32 | 33 |

*p < 0.05, **p < 0.01, ***p < 0.001

**주요 발견**:
- **Anthropic Haiku 4.5**가 가장 높은 상관관계 (r=0.631, p<0.001)
- **OpenAI GPT-5 mini**가 가장 낮은 오차 (MAE=5.06)
- Gemini는 샘플 수가 적고 상관관계가 낮음

#### 7.6.2 평가 영역별 AI-교사 일치도

| 평가 영역 | OpenAI r | Anthropic r | Gemini r | 평균 r |
|----------|----------|------------|----------|--------|
| **질문 전문성 (A1)** | **0.692*** | 0.612*** | 0.595*** | **0.633** |
| **응답 체계성 (B2)** | **0.671*** | **0.663*** | 0.602*** | **0.645** |
| 응답 맞춤화 (B1) | 0.496** | 0.592*** | 0.422* | 0.503 |
| 질문 구조화 (A2) | 0.394** | 0.524*** | 0.387* | 0.435 |
| **맥락 총점 (C1+C2)** | **0.133** | **0.234** | **0.111** | **0.159** |
| 학습 지원 (C2) | 0.051 | 0.030 | 0.034 | 0.038 |

***p < 0.001, **p < 0.01, *p < 0.05

**핵심 발견**:
1. **AI가 잘 평가하는 영역**: 질문 전문성, 응답 체계성 (r > 0.6)
2. **AI가 약한 영역**: 맥락 평가, 학습 지원 (r < 0.2)
3. **교사의 독특한 역할**: 대화 맥락과 학습 성장 지원에 대한 평가는 AI로 대체 불가

#### 7.6.3 AI-교사 평가 차이의 의미

**교사가 더 중요하게 보는 것**:
- **대화의 맥락과 연속성** (C1: 대화 일관성, r<0.3)
- **학습 성장 지원** (C2: 학습 지원, r<0.1)
- **교육적 맥락 고려** (A3: 맥락 적용)

**AI가 잘 평가하는 것**:
- **즉각적 품질 평가** (질문 전문성, 응답 체계성)
- **구조화된 기준** (명확한 루브릭 항목)
- **객관적 요소** (용어 정확성, 논리 구조)

**결론**:
1. **삼각 검증 완료**: 학생-교사-AI 모두 에이전트의 교육적 효과 확인

3. **교육적 가치와 실천의 일치**: 학생들이 선호하는 이유(사고 유도)와 교사가 높게 평가하는 이유(학습 지원) 일치

본 연구는 질문 명료화를 지원하는 AI Agent 시스템 MAICE를 실제 고등학교 수학 교실에 배포하여 운영한 결과를 제시하였다.

**주요 발견 사항**:

1. **명료화 프로세스의 활발한 작동**: 83.3%의 세션에서 명료화 질문을 수행하였으며, 명료화 있는 세션은 메시지 수가 2.4배 많아 더 깊은 학습 상호작용이 발생하였다.

2. **차별적 강점의 발견**: 총점에서는 차이가 없었으나, Agent는 C2(학습 지원)에서, Freepass는 A3(학습 맥락)에서 각각 우세하였다. 이는 두 모드가 서로 다른 교육적 강점을 가짐을 보여준다.

3. **하위권 학생에 대한 강력한 효과**: 
   - AI 모델 분석: Claude와 GPT-5 모델에서 하위권 학생에 대한 Agent의 C2(학습 지원) 효과 확인 (p<0.01)
   - **교사 평가 검증**: 중간고사 하위권(Q1) 학생이 에이전트 모드 사용 시 **8.00점 더 높은 평가** (p=0.029, Cohen's d=1.323)
   - 특히 응답 맞춤화(+3.86점)와 맥락 연결(+2.57점) 영역에서 큰 차이
   - Agent는 사고 과정 유도와 이해도 확인에서 특히 효과적

4. **다중 검증을 통한 신뢰성 확보**:
   - **다중 평가자**: 3개 독립 AI 모델 (ICC=0.595, Cronbach's α=0.840)
   - **교사 평가**: 전체적으로 에이전트 +5.36점 우세 (p=0.004, d=0.821)
   - **학생 선호도**: 59.3%가 에이전트 선호 (사고력 향상, 기억에 오래 남음)
   - **삼각 검증**: 학생-교사-AI 모두 에이전트의 교육적 효과에 합의

5. **AI-교사 평가 비교를 통한 각 주체의 역할 규명**:
   - AI는 질문 전문성(r=0.63)과 응답 체계성(r=0.65) 평가에 강점
   - 교사는 맥락 평가(r<0.2)와 학습 지원(r<0.1) 평가에 독보적
   - 상호 보완적 관계로 평가 품질 향상 가능

**이론적 함의**:

명료화 프로세스는 학생의 사고를 체계적으로 지원하며, 특히 학업 하위권 학생에게 효과적이다. 그러나 총점 비교만으로는 교육적 효과의 실체를 파악하기 어려우며, 세밀한 항목별 분석을 통해 차별적 효과를 규명할 수 있다.

**실천적 함의**:

1. **학생 수준별 차별화된 모드 적용**:
   - **하위권 학생 (중간고사 ~ 41점)**: 에이전트 모드 **강력 권장** (교사 평가 +8.00점, p=0.029)
   - **중상위권 학생**: 학생 선호도 및 학습 상황에 따라 유연하게 선택

2. **교육적 가치 중심의 설계**:
   - 학생 59.3%가 에이전트의 "생각하게 만드는" 과정을 선호
   - 교사도 에이전트의 응답 맞춤화(+1.09점), 학습 지원(+0.81점) 영역을 높게 평가
   - 단순한 효율성보다 학습 효과를 우선하는 설계 정당화

3. **AI-교사 협력 평가 시스템**:
   - AI: 질문·응답 품질의 즉각적·객관적 평가 (자동화 가능)
   - 교사: 맥락·성장 지원의 심층적·맥락적 평가 (인간 고유)
   - 상호 보완적 역할로 평가 효율성과 품질 동시 확보
