## 🎓 중요 참고사항

### Q1. 분석 스크립트가 에러나면?

**A1**: 다음을 확인하세요
1. 파일 경로가 맞는지 (`Gemini_루브릭채점_after_20251020_20251102_145330.json`)
2. 패키지가 모두 설치되었는지 (`pip list | grep factor`)
3. Python 버전이 3.8 이상인지 (`python --version`)

### Q2. Cronbach's α가 0.70 미만이면?

**A2**: 문제없습니다!
- 교육 연구에서는 α ≥ 0.65도 수용 가능
- 논문에서 "탐색적 도구이며, 추후 정교화 필요"로 기술
- 낮은 α의 원인 분석 (예: 문항 수 적음, 다차원성)

### Q3. 효과 크기가 작으면 (d < 0.2)?

**A3**: 오히려 정직한 보고!
- "작은 효과 크기도 교육 현장에서는 의미 있음" 강조
- 실용적 의의(practical significance) vs 통계적 유의성
- 장기 누적 효과 가능성 논의

### Q4. IRB 승인을 받지 않았다면?

**A4**: 두 가지 방안
1. **소급 승인**: 대부분 대학 IRB는 소급 심의 가능 (긴급 신청)
2. **면제 사유**: 정규 교육 과정 중 수행, 익명화 처리 → 면제 가능성
3. **논문에 명시**: "본 연구는 정규 수업의 일환으로 수행되어 IRB 면제 대상임"

## 💡 핵심 전략

### 심사자가 가장 주목할 3가지

1. **루브릭 신뢰도** (Cronbach's α)
   - 반드시 계산하고 보고
   - 0.70 이상이면 Best, 0.65 이상이면 OK
   - 낮더라도 정직하게 보고하고 한계점 인정

2. **효과 크기** (Cohen's d)
   - 모든 주요 비교에 추가
   - 유의하지 않아도 d 값은 보고
   - "작은 효과 크기도 현장에서는 의미"로 해석

3. **재현성** (프롬프트 공개)
   - GitHub + DOI 필수
   - 핵심 프롬프트는 논문 부록에도 포함
   - "완전히 투명하게 공개"가 강점
